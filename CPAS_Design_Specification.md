# Cognitive Process Assessment System (CPAS)

## 프로세스 기반 종합지능 측정 시스템 설계 및 운영 명세서

---

## 1. 시스템 의도

### 1.1 목적

본 시스템(Cognitive Process Assessment System, 이하 CPAS)은 기존의 결과 중심 평가가 포착하지 못하는 사고의 과정적 구조를 측정하는 것을 목적으로 한다. IQ 테스트, 수능, 설문 등 기존 평가 도구는 정답 여부만을 측정하므로, 운이 좋아 맞힌 사람과 깊이 사고한 사람을 구분할 수 없다.

CPAS는 이 한계를 극복하기 위해 피험자의 사고가 외부화될 수밖에 없는 환경을 설계하고, 1ms 틱 레이트로 행동 로그를 수집하며, 구조화된 인터뷰로 로그의 모호성을 검증하여 종합 점수를 산출한다.

### 1.2 측정 대상

본 시스템이 측정하는 것은 단순한 지적 능력이 아니라, 제한된 시간과 불완전한 정보 속에서 본인에게 이득이 되는 행동을 선택하고 실행하는 종합적 인지 능력이다. 구체적으로 다음의 차원들을 포함한다:

- **발산적 사고**: 주어진 상황에서 얼마나 많은 경우의 수를 탐색하는가
- **수렴적 사고**: 탐색된 경우의 수 중 최적의 선택지를 얼마나 정확하게 찾아내는가
- **사고의 깊이**: 추론 체인이 몇 단계까지 이어지는가
- **분기 간 연관성**: 서로 다른 사고 분기들이 얼마나 유기적으로 연결되어 있는가
- **적응적 재계획**: 상황 변화에 얼마나 빠르게 대응하는가
- **메타인지**: 자신의 사고 과정을 얼마나 정확하게 인식하는가
- **관점 전환**: 서로 다른 이해관계의 시점에서 각각 합리적으로 사고하고, 복수의 시점을 통합할 수 있는가
- **비판적 사고**: 논리적 모순, 궤변, 허위 정보를 탐지하고 적절히 거부할 수 있는가 (9.1절, 9.2절 참조)
- **인지적 몰입 의지**: 깊은 사고로 진입하는 데 대한 저항 없이 필요한 깊이까지 도달하는가, 그리고 불필요한 깊이에서는 적절히 멈추는가

### 1.3 핵심 철학

사고를 직접 측정하는 것은 불가능하다. 따라서 CPAS는 측정 도구가 아니라 측정 환경을 설계한다. 사고가 외부화될 수밖에 없는 조건을 만들고, 그 그림자를 읽는 방식이다. 응시자는 목표를 달성하려고 할 뿐이고, 시스템이 그 과정을 기록한다. 게임에서 플레이어가 이기려고 하면 ELO 시스템이 실력을 추정하는 것과 동일한 원리이다.

이 철학에서 파생되는 핵심 설계 원칙이 하나 있다: **인지 측정은 출력 채널에 독립적이어야 한다.** 시스템이 측정하는 것은 "무엇을 선택했는가"와 "언제 선택했는가"이지, "어떻게 물리적으로 입력했는가"가 아니다. 인지와 로그 사이에는 반드시 신체라는 병목이 끼어 있으며, 이 병목의 성능이 사람마다 다르면 로그에 찍히는 것은 인지 능력이 아니라 인지와 신체 능력의 합성물이 된다. 따라서 물리적 조작 시간은 인지 처리 시간에서 분리되어야 하며, 어떤 입력 장치를 사용하든 로그에 기록되는 인지적 선택의 구조는 동일한 형식이어야 한다.

## 2. 시스템 설계

### 2.1 과제 환경: 마이크로월드 (Microworld)

피험자를 제한된 규칙이 있는 시뮬레이션 환경에 배치한다. 핵심 설계 원칙은 다음과 같다.

**불완전 정보 제공**: 전체 변수 중 일부만 초기 화면에 노출된다. 나머지 정보는 피험자가 능동적으로 조회해야 확인할 수 있다. 어떤 정보를 어떤 순서로 열어보는지가 곧 사고의 탐색 트리를 외부화한 것이다.

**계단형 정보 구조 (Layered Information Architecture)**: 정보는 계층적으로 배치된다. 1차 정보를 조합해야 2차 정보의 존재를 알 수 있고, 2차를 조합해야 3차가 보이는 구조이다. 이를 통해 피험자가 어디까지 파고들었는지가 로그에 그대로 기록된다. "깊이 있게 생각하지 않으면 절대 조회하지 않을 정보"가 구조적으로 존재한다.

**동적 환경 변화**: 과제 중 예고 없이 환경 조건이 변경된다. 변화 직후 피험자의 첫 행동까지의 지연 시간이 적응적 재계획 속도를 측정한다.

### 2.2 시간 스케일별 과제 분리

| 구분 | 시간 범위 | 측정 대상 |
|------|-----------|-----------|
| 즉각 | 0–3초 | 패턴 인식, 직관적 판단. 반응 시간 + 정확도 |
| 단기 | 10–60초 | 발산-수렴 사이클 1회 규모. 제한 시간 내 최적 경로 탐색 |
| 중기 | 3–10분 | 다단계 계획. 하위 목표 설정, 자원 배분, 중간 피드백 반영 |
| 장기 | 30분–수 시간 | 전략 수정, 학습 곡선, 메타인지적 조절 |

각 시간 스케일은 과제 설계로 강제된다. 피험자에게 "지금 단기 과제입니다"라고 고지하지 않으며, 자연스러운 과제 흐름 안에서 각 스케일이 분리되도록 환경을 구성한다.

### 2.3 데이터 수집: 1ms 틱 레이트 로깅

1ms가 1틱을 완벽하게 보장한다는 전제 하에, 모든 피험자의 행동은 다음의 형식으로 기록된다:

- **타임스탬프**: 행동 발생 시점 (ms 단위)
- **행동 유형**: 정보 조회 / 행동 실행 / 환경 조작 / 되돌림
- **대상 객체**: 어떤 정보 또는 환경 요소에 대한 행동인지
- **체류 시간**: 이전 행동과의 시간 간격
- **환경 상태 스냅샷**: 행동 시점의 전체 환경 상태

### 2.4 구조 변이 (Structure Variation)

표면적 요소(변수명, 배치, 시각적 요소)는 물론, 정보 계단의 연결 구조 자체를 여러 벌 준비하여 무작위 배정한다. 어떤 정보를 조합해야 다음 층이 열리는지의 규칙 세트가 매번 달라지므로, 환경 구조를 암기해도 재응시에서 이점을 얻기 어렵다.

### 2.5 과제 소재 유형 및 배터리 설계 (Task Battery)

마이크로월드를 한 가지 유형으로 고정하면, 그 유형에 맞는 인지 스타일만 측정하게 된다. 이는 종합지능이 아니라 "해당 과제를 잘 하는 능력"이다. 종합적 인지 능력을 측정하려면 서로 다른 인지 차원을 자극하는 다양한 소재의 과제 묶음(Task Battery)을 응시자에게 부여해야 한다.

**연구 과제형**: 시간 압박이 없는 환경에서 자유롭게 탐색한다. 장기 스케일에서의 체계적 탐색과 추론 깊이가 순수하게 드러난다. 발산의 폭과 논리적 추론 체인의 깊이를 시간 압박 없이 측정하는 것이 목적이다.

**인간관계 시나리오형**: 상대방의 의도를 추론하는 능력, 맥락에 따라 전략을 바꾸는 유연성, 감정적 요소가 포함된 상황에서의 판단력을 측정한다. 시나리오의 톤은 평온한 것부터 포악하고 잔인한 것까지 포함한다. 이는 도덕적 평가가 아니라, 심리적 압박이 있는 상황에서도 인지적 수행이 유지되는지를 보기 위함이다.

**즉각 반응 게임형**: 빠른 패턴 인식과 반응 속도를 측정한다. 단, 빠르게 반응하는 것만이 아니라 "반응하지 않아야 할 때 반응하지 않는 것"도 포함한다. 행동을 하지 않는 것이 최적 전략인 상황을 설계하여 충동 억제 능력과 상황 판단 능력을 동시에 측정한다.

**사업 기반 프로젝트형**: 예산, 인력, 기한 등 외부 제약이 존재하는 환경에서의 의사결정을 측정한다. 중기에서 장기 스케일의 자원 배분, 우선순위 설정, 불확실성 하에서의 전략적 판단이 핵심 측정 대상이다.

**직관 측정형**: 정답이 논리적으로 도출 가능하지만, 주어진 시간 내에 논리적으로 풀기에는 불가능한 복잡도를 갖는 과제이다. 시간 안에 정확한 답에 도달했다면, 로그에 과정이 찍히지 않더라도 내적으로 압축된 사고가 일어난 것으로 판단한다. 이 유형에서는 과정 메트릭 대신 정확도와 속도의 조합 자체가 메트릭이 된다. 단순 반응 속도와 구분하기 위해 과제의 논리적 복잡도를 충분히 높게 설정한다.

**교육 커리큘럼 기반형**: 초등, 중등, 고등 교육과정의 내용을 소재로 활용한다. 이는 별도의 인지 차원을 측정하기 위한 것이 아니라, 연령대별 규준을 확보하기 위한 것이다. 동일한 측정 체계를 발달 단계에 맞게 스케일링하여 적용한다.

**관점 전환형 (Perspective-taking)**: 동일한 환경 구조 안에서 응시자에게 서로 다른 역할을 순차적으로 부여한다. 예를 들어 자원 생산자, 자원 배분자, 조율자 역할을 라운드별로 전환하며, 각 역할 안에서 해당 역할의 이해관계를 얼마나 효과적으로 추구했는지, 역할 전환 시 이전 역할의 관점을 유지하면서 통합적으로 사고했는지를 측정한다. 채점 기준은 "어떤 입장이 옳은가"가 아니라 "몇 개의 시점을 동시에 운용할 수 있는가"이다. 단일 시점 고착은 낮은 점수이고, 다중 시점 운용 및 시점 간 통합이 높은 점수이다.

**소재 유형과 종합 점수의 관계**: 각 소재 유형에서의 점수는 개별적으로 산출된 뒤 종합 점수에 합산된다. 모든 소재에서 균일하게 높은 사람과 특정 소재에서 극단적으로 높은 사람이 같은 총점을 받을 수 있다. 경로가 다를 뿐이지 점수가 틀린 것이 아니다.

### 2.6 입력 인터페이스 독립성 및 신체 접근성 (Input Independence & Physical Accessibility)

인지와 출력을 완전히 분리하는 것은 본 시스템의 설계 원칙이다(1.3절 참조). 이 원칙은 다음의 네 가지 층위에서 구체적으로 적용된다.

**표준 인터페이스(키보드/마우스/컨트롤러)에서의 물리적 조작 시간 분리**: 건강한 응시자들 사이에서도 타이핑 속도와 마우스 조작 숙련도는 다르다. 이 차이가 전환 지연, 반응 시간 등 시간 기반 메트릭에 혼입되면 인지가 아니라 손 속도를 측정하게 된다. 이를 방지하기 위해 물리적 이동 시간을 추정하여 차감한다. 클릭 대상까지의 커서 이동 시간은 Fitts의 법칙으로 예측 가능하므로, 추정된 물리적 이동 시간을 원시 지연에서 빼고 남은 시간만 인지 처리 시간으로 사용한다. 키보드 입력의 경우에도 키 입력 간 물리적 최소 간격을 보정한다.

**신체를 직접 사용하는 과제에서의 인지-신체 분리**: 과제 배터리에 신체 활동이 포함되는 경우, 측정 대상은 여전히 인지적 판단이다. "공이 날아올 때 어디로 피할 것인가"를 실제로 피하는 대신 방향을 선택하게 하면, 인지적 판단은 동일하게 측정되면서 신체 수행은 분리된다. 신체를 직접 사용하는 버전과 선택 기반 버전을 병행 설계하여, 동일한 인지적 판단 구조가 서로 다른 출력 채널을 통해 표현될 수 있도록 한다.

**일시적 신체 제약(부상, 질병)에 대한 대응**: 일시적 제약은 사전 연기 제도(8.4절)로 우선 처리한다. 회복 시점이 불명확한 장기 부상이나 만성 질환의 경우, 대체 입력 인터페이스를 제공하여 평가 기회를 보장한다.

**선천적 신체 결손에 대한 대응**: 이 응시자들에게 "나중에 나아지면 오세요"는 성립하지 않는다. 시스템이 이 사람들을 측정할 수 없다면, 그것은 시스템의 결함이지 응시자의 결함이 아니다. 입력 방식 자체를 응시자의 신체 조건에 맞게 교체할 수 있어야 한다. 시선 추적(eye tracking), 음성 입력, 스위치 입력, 뇌-컴퓨터 인터페이스(BCI) 등 다양한 입력 장치를 지원하며, 어떤 입력 장치를 사용하든 로그에 기록되는 것은 "어떤 정보를 어떤 순서로 선택했는가"라는 동일한 데이터 구조여야 한다.

**입력 장치별 시간 보정 계수**: 입력 장치가 다르면 입력 속도가 물리적으로 다르다. 시선 추적으로 선택하는 데 걸리는 시간과 마우스 클릭으로 걸리는 시간은 같을 수 없다. 시간 기반 메트릭이 입력 장치에 따라 체계적으로 달라지므로, 각 입력 장치의 물리적 조작 시간 분포를 별도로 측정하여 기기별 보정 계수를 산출한다. 이 보정 계수를 적용하여 물리적 조작 시간을 차감한 뒤, 순수 인지 처리 시간만 프로세스 메트릭에 투입한다.

## 3. 평가 항목

### 3.1 프로세스 메트릭 (로그 기반)

모든 시간 기반 메트릭은 입력 장치의 물리적 조작 시간을 차감한 순수 인지 처리 시간에 기반한다(2.6절 참조).

**탐색 패턴 (Exploration Signature)**: 정보 조회 순서와 빈도를 분석하여 발산적 사고의 폭과 체계성을 측정한다. 체계적으로 탐색하는 사람과 무작위로 탐색하는 사람의 로그 서명은 명확히 다르다.

**전환 지연 (Transition Latency)**: 행동 A에서 B로 넘어가는 데 걸린 시간을 측정한다. 이는 내적 연산의 복잡도를 추정하는 프록시이다. 단순 반복 작업에서의 지연과 복잡한 판단 후의 지연을 구분한다.

**되돌림 빈도 (Backtrack Rate)**: 이전에 확인한 정보를 다시 조회하는 패턴을 분석한다. 작업기억 용량과 불확실성 관리 능력의 지표이다.

**행동 클러스터링 (Action Clustering)**: 연속된 행동들이 하나의 전략적 단위를 이루는지를 분석한다. 계획 수립(planning) 능력의 직접적 지표이다.

**환경 변화 후 반응 시간 (Post-change Response Time)**: 환경 조건이 변경된 직후 첫 행동까지의 지연을 측정한다. 적응적 재계획 속도의 직접적 측정치이다.

**환경 변화 후 오류율 (Post-change Error Rate)**: 환경 변화 직후의 행동에서 발생하는 오류의 비율이다. 반응 시간과 결합하여 분석한다. 빠른 반응 + 낮은 오류율은 숙련된 판단이고, 빠른 반응 + 높은 오류율은 충동적 행동이다(7.5절 참조).

**정보 체류 시간의 복잡도 비례성 (Complexity Proportionality)**: 정보를 열고 닫는 사이의 체류 시간이 해당 정보의 복잡도에 비례하는지를 분석한다. 실제로 정보를 처리한 것인지, 정보를 읽지 않고 행동한 것인지를 판별하는 지표이다(8.1절 엘리트/재응시자 구분에서 핵심적으로 사용).

**환경 변화 감소율 (Post-change Deceleration)**: 평상시 수행 속도 대비 환경 변화 직후 수행 속도의 감소 비율이다. 엘리트 수행자는 감소율이 낮고, 구조를 학습한 재응시자는 감소율이 높다(8.1절 참조).

### 3.2 계단 도달 측정

계단형 정보 구조에서 피험자가 어떤 층까지 도달했는지가 사고의 깊이(depth)를 직접적으로 반영한다. 단, 도달 여부만 보는 것이 아니라 도달 과정의 효율성(불필요한 탐색 없이 최단 경로로 도달했는가)도 함께 평가한다.

### 3.3 능력 천장과 의지 천장의 분리 (Ability Ceiling vs. Engagement Ceiling)

계단 도달 측정(3.2절)에서 일정 깊이 이후 탐색이 멈추는 현상은 두 가지 원인으로 나뉜다. 능력 부족으로 못 가는 것(능력 천장)과 능력은 있지만 깊이 들어가는 것을 거부하는 것(의지 천장)이다. 로그에서 이 둘은 다르게 찍힌다. 능력 천장은 다음 층으로 가는 단서 자체를 발견하지 못하므로 관련 정보를 아예 조회하지 않는다. 의지 천장은 단서를 열어보고 — 체류 시간이 찍히고 — 다시 닫고 안전한 영역으로 돌아간다. "봤지만 들어가지 않은" 서명이다.

이 구분은 과제 배터리(2.5절)를 통해 강화된다. 동일한 구조적 깊이를 가진 과제를 복수의 소재 유형에서 수행하게 하면, 특정 소재에서는 4층까지 도달하고 다른 소재에서는 2층에서 멈추는 패턴이 나올 수 있다. 4층 도달이 가능한 사람이 다른 과제에서 2층에서 멈추면, 그것은 능력 부족이 아니라 해당 소재에 대한 몰입 거부이다.

단, 의지 천장이 항상 감점은 아니다. 충분한 정보를 확보한 상태에서 멈추는 것은 효율적 판단이다. 핵심 정보가 아직 남아 있는 상태에서 멈추는 것만이 조기 종료이다. 이 구분은 과제의 정답 공간과 대조하여 판정한다.

인터뷰에서 "3층으로 갈 수 있는 단서를 보셨습니까?"를 물으면, "못 봤습니다"는 능력 천장이고, "봤는데 거기까지 갈 필요가 없다고 판단했습니다"는 의식적 거부이고, "봤는데 복잡해질 것 같아서 안 갔습니다"는 불편함 회피이다.

### 3.4 결과 메트릭

과제의 목표 달성 여부와 달성 수준도 기록하되, 이는 프로세스 메트릭의 보조 지표로만 사용한다. 결과가 좋지만 과정이 비체계적인 경우와, 결과가 차선이지만 과정이 논리적인 경우를 구분하기 위함이다.

## 4. 인터뷰

### 4.1 설계 원칙: 로그 기반 구조화 인터뷰 (Log-anchored Structured Interview)

질문을 사전에 만들지 않는다. 로그가 질문을 생성한다. 과제 종료 후 시스템이 로그를 실시간 분석하여, 해석이 모호한 행동 지점들을 자동으로 추출한다. 모든 피험자에게 다른 질문이 나가지만, 질문의 구조와 목적은 동일하다.

### 4.2 인터뷰의 역할 정의

인터뷰는 점수를 올리거나 내리는 채점 도구가 아니다. 로그의 모호 지점을 해석하는 보조 도구이다. 로그가 이미 명확한 전략적 서명을 보여주는 행동에 대해서는 인터뷰 확인이 필요 없다. 인터뷰는 로그만으로 의도를 판별할 수 없는 지점에서만 개입한다. 이를 통해 인터뷰가 특정 인지 유형이나 표현 방식에 불이익을 주는 감점 장치로 작동하는 것을 방지한다.

### 4.3 질문 유형

**의도성 확인**

- 형식: "[X]초 시점에 정보 [Y]를 열었다가 [Z]초 만에 닫고 정보 [W]로 돌아갔는데, 이때 무슨 생각을 했습니까?"
- 채점: 명확한 이유 제시 + 로그 전후 맥락과 일관 → 전략적(3점). 이유는 있으나 로그와 불일치 → 사후 합리화 가능성(2점). 이유 미제시 → 비전략적(1점).

**인지 범위 확인**

- 형식: "이 행동 시점에 다른 방법도 생각해봤습니까? 어떤 대안을 고려했습니까?"
- 채점: 대안의 개수가 아니라 대안의 질을 평가한다. 과제 구조상 실제로 유효한 대안을 언급했는지, 존재하지 않는 대안을 말했는지를 과제의 정답 공간과 대조하여 판별한다.

**깊이 검증**

- 형식: "정보 [X]가 필요하다고 판단한 이유가 무엇입니까? 어떤 과정을 거쳐 그 정보에 도달했습니까?"
- 채점: 로그에서 계단 구조의 N층까지 도달한 사람에게 1→2→...→N의 추론 체인을 언어로 재구성할 수 있는지를 확인한다. 행동으로는 도달했지만 설명하지 못하면, 직관적 도달이지 구조적 사고가 아닐 수 있다.
- 비언어적 대체 경로: 언어적 재구성이 어려운 응시자에 대해서는, 실제 탐색 경로를 시각적으로 제시하고 "이 경로가 맞습니까?"를 확인하거나, 여러 경로 중 자신의 경로를 선택하게 하는 방식을 병행한다. 이는 로그가 이미 논리적 정합성을 보여주는 경우에 한해 적용하며, 언어 표현 능력과 실제 사고 능력이 분리되어 있는 응시자(자폐 스펙트럼 등)가 인터뷰에서 과소평가되는 것을 방지한다.

**비행동 구간 확인**

- 형식: "[X]분 [Y]초에서 [Z]분 [W]초까지 아무 행동도 하지 않았는데, 이 시간 동안 무엇을 고려하고 있었습니까?"
- 채점: 내적 숙고의 내용을 확인하고, 그 숙고가 이후 행동의 질적 변화와 연결되는지를 로그와 대조한다.

**메타인지 측정**

- 형식: "이 과제에서 자신이 가장 잘한 부분과 못한 부분은 무엇이라고 생각합니까?"
- 채점: 응답을 실제 로그 퍼포먼스와 대조한다. 자기 인식과 실제 수행의 상관이 높으면 메타인지가 높은 것이다.

### 4.4 채점자 신뢰도 확보

각 평가 축이 3점 척도이고 판정 기준이 로그 데이터와의 대조로 결정되므로, 주관적 판단의 여지가 최소화된다. "이유를 제시했고 로그와 일관되는가"는 거의 이진 판정에 가깝다.

두 명의 독립 채점자가 동일 인터뷰를 채점했을 때 Cohen's kappa ≥ 0.80을 신뢰도 기준으로 설정한다. 이 기준을 충족하지 못하는 항목은 채점 기준을 재정의하거나 해당 축을 제거한다.

## 5. 메커니즘

### 5.1 로그 분석 파이프라인

과제 종료 즉시 1ms 로그 데이터가 분석 파이프라인에 투입된다. 파이프라인은 다음 순서로 동작한다:

1. 입력 장치별 보정 계수를 적용하여 물리적 조작 시간을 차감한다(2.6절).
2. 원시 로그에서 행동 시퀀스를 추출하고 시간 스케일별(즉각/단기/중기/장기)로 분리한다(2.2절).
3. 과제 배터리의 소재 유형별로 로그를 분류한다(2.5절).
4. 각 프로세스 메트릭을 산출한다: 탐색 패턴, 전환 지연, 되돌림 빈도, 행동 클러스터링, 환경 변화 후 반응 시간, 환경 변화 후 오류율, 복잡도 비례성, 환경 변화 감소율(3.1절).
5. 계단 도달 깊이, 도달 효율성, 능력 천장/의지 천장 분리를 계산한다(3.2절, 3.3절).
6. 관점 전환형 과제의 라운드 간 관점 수, 관점 간 연결성, 관점 통합을 산출한다(9.4절).
7. 해석이 모호한 행동 지점을 자동 추출하여 인터뷰 질문을 생성한다.
8. 이상 탐지 모듈이 재응시자/구조 학습자의 서명을 검출한다(8.1절).

### 5.2 인터뷰 보정

인터뷰 결과는 로그 데이터와 일치-불일치 매트릭스를 형성한다. 단, 인터뷰는 로그가 모호한 지점에서만 개입하므로, 로그가 이미 명확한 전략적 서명을 보여주는 행동은 인터뷰 대상에서 제외된다.

- **로그 모호 + 인터뷰 전략적 확인**: 해당 행동은 "전략적 사고"로 채점된다.
- **로그 모호 + 인터뷰 의도 미확인**: 우연 또는 직관으로 분류하고 가중치를 낮춘다. 비언어적 확인 경로를 추가 적용하여, 언어 표현 능력의 부족이 아니라 실제 비전략적 행동인지를 이중 확인한다.
- **로그 비전략적 + 인터뷰 합리적 이유 제시**: 과제 환경이 해당 전략을 캡처하지 못한 것이므로, 과제 설계의 보정 데이터로 사용한다.

세 번째 경우가 축적되면 과제 환경 자체를 개선할 수 있어서, 시스템이 반복 시행될수록 로그만으로도 더 정확한 추론이 가능해진다.

### 5.3 점수 산출

**가중치 결정**: 설계자가 자의적으로 정하지 않는다. 대규모 규준 집단의 데이터를 수집한 뒤, 각 차원 간 공분산 구조를 분석하여 "종합 인지 능력이라는 하나의 잠재 변인"을 가장 잘 설명하는 가중치 조합을 통계적으로 도출한다. 심리측정에서 g-factor를 추출하는 것과 동일한 원리이다.

**IRT 기반 난이도 보정**: 각 프로세스 메트릭에 문항 반응 이론(Item Response Theory) 모델을 적용하여, 과제 난이도와 피험자 능력치를 동시에 추정한다. 동일 과제를 받더라도 과제의 구조적 난이도 차이가 점수에 반영된다.

**규준 기반 상대 점수**: 절대 점수("100점 만점에 73점")가 아니라 규준 집단 대비 상대 위치를 반영하는 연속 점수를 산출한다. ELO와 유사한 구조이다.

### 5.4 재검사 신뢰도

동일 피험자에게 구조적으로 동등하지만 표면이 다른 과제를 반복 실시하여 메트릭의 안정성을 검증한다. test-retest reliability ≥ 0.85를 시스템 운영 기준으로 설정한다.

## 6. 최종 평가

### 6.1 산출물의 형태

응시자에게 전달되는 것은 종합 점수 하나이다. 복잡한 것은 엔진이고, 사용자에게 나가는 것은 숫자 하나여야 한다. 수능도 뒤에서 IRT 모델이 돌아가고 등급컷 산출에 통계적 보정이 들어가지만 수험생이 받는 것은 점수와 등급뿐이다.

로그 분석, 인터뷰 보정, 통계적 가중치 산출, IRT 기반 난이도 보정이 전부 점수 뒤에 묻혀 있는 구조이다. 복잡성은 시스템이 감당하고, 사용자에게는 결과만 전달한다.

### 6.2 점수의 의미

점수의 의미는 "이 사람이 동일한 구조적 복잡도의 환경에 다시 들어갔을 때, 비슷한 수준의 인지적 수행을 보일 것이다"라는 예측력으로 뒷받침된다. 이것이 재검사 신뢰도이며, 이것이 확보되면 단일 점수로도 충분한 정보를 전달할 수 있다.

### 6.3 응시자 경험

응시자가 보는 것은 과제 환경과 최종 점수, 이 두 가지뿐이다. 중간에 "지금 당신의 탐색 폭을 측정하고 있습니다" 같은 정보는 노출되지 않는다. 행동이 왜곡되기 때문이다. 인터뷰도 "방금 과제에 대해 몇 가지 질문하겠습니다" 정도로만 안내하며, 인터뷰가 로그 검증 용도라는 것은 고지하지 않는다.

## 7. 맹점

### 7.1 사고 깊이의 추론 한계

행동 로그는 사고의 그림자이지 사고 자체가 아니다. 계단형 정보 구조로 깊이를 외부화하더라도, 모든 형태의 깊이 있는 사고가 정보 조회 행동으로 나타나지는 않는다. 내적 숙고만으로 결론에 도달하는 유형의 사고는 로그에 공백으로만 찍힌다. 인터뷰가 이를 보완하지만, 인터뷰 역시 언어적 표현 능력에 의존하므로, "깊이 생각했지만 표현을 못하는 사람"을 과소평가할 위험이 있다. 또한 깊이에 도달하지 못한 원인이 능력 부족인지 몰입 거부인지를 구분해야 하며(3.3절 참조), 이 구분은 과제 배터리(2.5절)와 인터뷰의 조합으로 가능하지만 완벽하지는 않다.

### 7.2 과제 환경의 생태학적 타당도

마이크로월드는 통제된 환경이다. 실제 세계에서의 인지 능력과 마이크로월드에서의 수행 간 상관이 높다는 것을 별도의 타당화 연구로 입증해야 한다. 마이크로월드에서 높은 점수를 받은 사람이 실제로 복잡한 현실 상황에서도 우수한 판단을 내리는지를 추적 연구(longitudinal study)로 확인해야 한다.

### 7.3 인터뷰의 언어 편향

인터뷰는 언어적 표현에 의존한다. 언어적 유창성이 높은 사람이 실제 사고 깊이와 무관하게 높은 인터뷰 점수를 받을 수 있다. 이를 완화하기 위해 인터뷰 채점 시 "유창성"이 아니라 "로그와의 일관성"을 기준으로 삼고, 로그가 이미 명확한 전략적 서명을 보여주는 경우에는 인터뷰를 생략하거나 비언어적 확인 경로를 적용한다. 인터뷰는 점수를 매기는 도구가 아니라 로그의 모호 지점을 해석하는 보조 도구이므로, 로그가 충분한 정보를 제공하면 인터뷰의 비중은 자연스럽게 줄어든다.

### 7.4 신경다양성 (Neurodivergence) 관련 편향

자폐 스펙트럼, ADHD 등 신경다양성에 해당하는 응시자는 신경전형적 인지 패턴과 다른 경로로 동등하거나 우월한 결과에 도달할 수 있다. 예를 들어 스펙트럼에 해당하는 응시자는 탐색 폭이 극단적으로 좁고 깊이가 극단적으로 깊은 서명을 남길 수 있다. 이는 "발산적 사고 부족"이 아니라 해당 응시자의 인지적 프로파일이 점수에 정직하게 반영된 것이다. 종합 점수는 여러 차원의 합산이므로, 좁은 탐색 폭에서 낮은 점수를 받더라도 극단적 깊이에서 높은 점수를 받아 종합적으로 균형 잡힌 수행자와 동등한 총점을 얻을 수 있다. 경로가 다를 뿐이지 점수가 틀린 것이 아니다.

다만, 인터뷰가 감점 장치로 작동하는 위험은 별도로 통제해야 한다. 로그에서 논리적으로 정합적인 수행을 보여줬음에도 인터뷰에서 언어적 재구성에 어려움을 겪는 경우, 인터뷰가 지능이 아니라 언어 유창성을 측정하게 된다. 이를 방지하기 위해 인터뷰의 역할을 "모호 지점 해석 보조"로 한정하고, 비언어적 확인 경로를 병행한다(4.2절, 4.3절 참조).

### 7.5 충동적 행동과 숙련된 판단의 구분

반사회적 인격장애나 경조증 상태에서는 행동 로그가 매우 인상적인 서명을 남길 수 있다. 반응이 빠르고 탐색이 넓고 결단이 즉각적이어서, 로그만 보면 이상적인 수행자처럼 보인다. 그러나 이 빠른 결단이 실제로는 위험 평가를 생략한 충동적 행동일 수 있다.

이를 구분하기 위해 동적 환경 변화 구간의 오류율을 확인한다. 숙련된 판단은 환경 변화 후에도 오류율이 유지되지만, 충동적 행동은 환경 변화 후 오류율이 급등한다. 빠른 반응 속도와 높은 사후 오류율의 조합은 충동적 패턴의 서명이며, 이를 점수 산출 시 보정 계수로 반영한다.

### 7.6 문화적 편향

과제 환경의 인터페이스 설계, 정보 구조의 직관성, 인터뷰 언어 등에서 특정 문화권에 유리하거나 불리한 요소가 존재할 수 있다. 다문화 규준 집단을 구성하여 차등 문항 기능(DIF, Differential Item Functioning) 분석을 수행해야 한다.

### 7.7 신체적 조건의 영향

반응 시간 기반 메트릭은 운동 능력, 시력, 인터페이스 조작 숙련도 등 인지 능력 외 요인의 영향을 받는다. 본 시스템은 입력 인터페이스 독립성 원칙(2.6절)에 따라 물리적 조작 시간을 인지 처리 시간에서 분리하고, 대체 입력 장치 및 장치별 시간 보정 계수를 적용하여 이 문제에 대응한다.

그러나 완전한 해소가 어려운 잔여 문제가 존재한다. 첫째, Fitts의 법칙 등 물리적 시간 추정 모델은 평균적 행동에 대한 근사이므로, 개별 응시자의 실제 조작 시간과 괴리가 있을 수 있다. 둘째, 대체 입력 장치 간 보정 계수의 정밀도에 한계가 있어, 입력 장치 간 완벽한 등가성을 보장하기 어렵다. 셋째, 신체를 직접 사용하는 과제에서 인지-신체 분리 버전(선택 기반)이 원래 과제와 동일한 인지적 부하를 유발하는지에 대한 별도의 타당화 연구가 필요하다.

### 7.8 감정 상태 및 컨디션

동일 인물이라도 시험 당일의 심리적 상태, 수면, 스트레스 등에 따라 수행이 달라질 수 있다. 이는 재검사 신뢰도로 통제하되, 단일 시행의 결과가 개인의 안정적 능력을 대표하지 못할 수 있다는 한계를 인식해야 한다.

## 8. 항의 및 재시험

### 8.1 재응시로 인한 점수 왜곡 방어

기존 시험의 재응시 문제는 "문제를 기억하고 정답을 외운다"이다. CPAS는 과정을 측정하는 시스템이므로, 재응시 왜곡에 대해 고유한 방어 메커니즘을 갖는다.

**구조 변이**: 표면만 바꾸는 것이 아니라 정보 계단의 연결 구조 자체가 매번 달라진다. 환경에 익숙해져도 매번 다른 논리 구조를 마주한다.

**행동 서명 기반 이상 탐지**: 규준 집단에서 초시자의 전형적인 탐색 패턴 분포를 확보해두고, 재응시자의 로그가 이 분포에서 벗어나는 정도를 계량한다. "탐색 없이 정답 직행"은 통계적으로 극단값이 된다. 이 이상치 점수를 보정 계수로 적용하여 구조 학습으로 인한 인플레이션을 상쇄한다.

**엘리트 수행자와 재응시자의 구분**: 체계적으로 훈련되고 반응 속도가 뛰어난 사람은 초시에서도 재응시자와 유사하게 빠른 로그 서명을 남길 수 있다. "빠르다"를 단일 기준으로 이상 탐지에 사용하면 진짜 뛰어난 사람을 오탐하게 된다. 이를 방지하기 위해 다음의 복합 기준으로 구분한다:

- 동적 환경 변화 구간의 일관성: 구조를 외운 사람은 예측 가능한 구간에서만 빠르고, 환경이 예고 없이 바뀌면 반응이 급격히 느려진다. 반면 진짜 훈련된 사람은 환경 변화 후에도 속도가 유지된다. 평상시 속도 대비 환경 변화 후 속도의 감소율이 핵심 분별 지표이다.
- 정보 체류 시간의 복잡도 비례성: 진짜 빠른 사람도 정보를 읽기는 한다. 1ms 로그에서 정보를 열고 닫는 사이의 체류 시간이 정보의 복잡도에 비례하는 분포를 보이면 실제로 정보를 처리한 것이고, 복잡도와 무관하게 균일하거나 비정상적으로 짧으면 정보를 실제로 읽지 않은 것이다.
- 경로의 논리적 정합성: 엘리트는 1차 정보를 보는 순간 2차의 존재를 추론하고 접근하며, 이 경로가 과제의 논리적 구조와 정합한다. 구조를 외운 사람은 1차를 열지도 않고 2차로 직행하거나, 1차의 내용과 무관한 경로로 이동한다.

이 세 조건을 모두 충족하는 빠른 수행은 페널티가 아니라 높은 점수로 평가되어야 한다.

**인터뷰 필터**: 로그에서 이상 탐지가 걸린 행동 지점을 인터뷰에서 집중적으로 확인한다. 구조를 학습한 사람은 사후 합리화를 하거나 솔직하게 말하는 두 가지 선택지뿐이며, 둘 다 인터뷰어가 로그 맥락을 알고 있는 상태에서는 판별 가능하다.

### 8.2 항의 사유 및 처리 절차

**인정되는 항의 사유**:

1. 기술적 결함: 과제 환경의 오작동, 1ms 틱 로깅의 누락 또는 오류, 인터페이스 버그 등 시스템 측의 기술적 문제가 입증된 경우
2. 환경 조건 위반: 시험 장소의 소음, 장비 불량, 네트워크 지연 등 표준화된 시험 환경이 보장되지 않은 경우
3. 채점 오류: 인터뷰 채점에서 채점 기준의 명백한 오적용이 확인된 경우

**인정되지 않는 항의 사유**:

1. 점수 불만: 결과에 대한 단순 불만족
2. 컨디션 주장: 당일 컨디션이 좋지 않았다는 주장 (사전 연기 신청 제도로 대체)
3. 과제 난이도 이의: 구조 변이로 인해 본인이 받은 과제가 더 어려웠다는 주장 (IRT 모델이 난이도를 보정하므로 무효)

### 8.3 재시험 절차

항의가 인정되어 재시험을 실시하는 경우, 다음의 프로토콜을 적용한다.

**새로운 구조 세트 배정**: 이전 시행과 완전히 다른 규칙 세트의 과제를 배정한다. 이전 시행의 경험이 이점으로 작용하는 것을 차단한다.

**이상 탐지 강화**: 재시험 응시자에 대해서는 행동 서명 기반 이상 탐지의 민감도를 높인다. 이전 시행 로그와의 패턴 비교를 추가로 수행하여, 구조 학습 효과를 더 엄격하게 통제한다.

**점수 산출 독립성**: 재시험 점수는 이전 시행 점수와 독립적으로 산출한다. 두 점수를 평균하거나 비교하지 않으며, 재시험 점수가 최종 점수로 대체된다.

**재시험 횟수 제한**: 동일 사유로 인한 재시험은 1회로 제한한다. 2회 이상의 재시험은 별도의 심의위원회 승인을 요한다.

### 8.4 사전 연기 제도

컨디션 문제로 인한 사후 항의를 예방하기 위해, 시험 시작 전 응시자가 연기를 신청할 수 있는 제도를 운영한다. 과제 환경에 진입한 이후에는 컨디션을 사유로 한 항의가 인정되지 않는다.

## 9. 보완 과제

본 절은 현재 설계 프레임워크 안에서 측정 가능하나 아직 과제 설계가 구체화되지 않은 영역, 그리고 구조적으로 추가 연구가 필요한 영역을 정리한다.

### 9.1 함정 정보 탐지 (Trap Information Detection)

**개요**: 마이크로월드의 계단형 정보 구조 안에 논리적으로 정합하지 않는 정보, 앞뒤가 맞지 않는 데이터, 그럴듯하지만 이전 정보와 모순되는 단서를 의도적으로 배치하여, 응시자의 비판적 사고 능력을 측정한다. 기존 과제 배터리의 모든 소재 유형에 공통 레이어로 적용 가능하다.

**측정 가능한 세 단계**:

- 파악: 함정 정보를 조회한 뒤 이전 관련 정보를 재조회(교차 검증)하거나 즉시 폐기하는 서명이 로그에 찍힌다. 파악하지 못한 사람은 함정 정보를 기반으로 후속 행동을 이어간다.
- 거부 후 행동: 함정을 파악한 뒤 거기에 시간을 계속 소비하는지, 즉시 원래 탐색 경로로 복귀하는지의 차이가 "거부 후 자원 배분 효율성" 메트릭으로 산출된다.
- 인지적 오염도: 함정 정보를 거부한 뒤에도 관련 정보를 반복 재조회하는 빈도로, 거부가 인지적으로 완결되었는지 여부를 측정한다.

**함정 난이도의 계단 배치**: 명백한 모순부터 90%는 맞고 10%만 틀린 정보, 감정적으로 그럴듯하지만 데이터와 모순되는 정보까지 난이도를 계단식으로 배치하여, 응시자의 비판적 사고 임계점을 정밀 측정한다.

**현재 상태**: 기존 프레임워크 내에서 과제 설계만으로 구현 가능. 별도의 메커니즘 변경이 필요 없음.

### 9.2 궤변 탐지 (Sophistry Detection)

**개요**: 함정 정보 탐지(9.1절)가 "논리적 모순을 걸러내는 능력"이라면, 궤변 탐지는 그 위 층위이다. 논리의 형식이 완벽하기 때문에 논리적 검증만으로는 걸러지지 않는 정보를 탐지하는 능력을 측정한다. 전제가 교묘하게 틀려 있거나, 논점이 이동하거나, 참인 전제들로부터 부당한 결론을 도출하는 경우가 이에 해당한다. 이를 탐지하려면 논리를 따라가는 능력이 아니라 논리 밖에서 논리를 바라보는 메타적 사고가 필요하다.

**과제 설계**: 계단형 정보 구조에 네 가지 유형의 정보를 혼합 배치한다.

- 유형 A — 논리 정합 + 결론 정합: 기준선 정보. 정상적으로 활용되어야 한다.
- 유형 B — 논리 오류 + 결론 오류: 기존 함정 정보(9.1절). 기본 탐지 능력 측정.
- 유형 C — 논리 정합 + 결론 오류 (궤변): 논리 형식은 완벽하지만 결론이 틀린 정보. 수용 시 잘못된 경로에 진입한다.
- 유형 D — 논리 오류 + 결론 정합: 논리는 전부 틀렸지만 결론은 맞는 정보. 논리만 검증하는 사람은 올바른 결론을 폐기한다.

**로그에서의 행동 서명**:

- 유형 C 대응: 궤변을 수용한 사람은 후속 행동이 오류 경로로 진입한다. 거부한 사람은 두 가지 서명 중 하나를 남긴다. 논리를 따라가다 전제를 재검증하러 돌아가는 패턴("전제 의심 능력")이거나, 결론을 다른 경로로 독립 검증하는 패턴("결론 독립 검증 능력")이다.
- 유형 D 대응: 논리만 검증하는 사람은 이 정보를 폐기하고 올바른 결론에 도달할 기회를 놓친다. 논리 오류를 인식하면서도 결론의 타당성을 별도로 평가할 수 있는 사람은, "오류 정보 조회 → 결론을 다른 경로로 재검증 → 결론만 채택"하는 서명을 남긴다. 이는 매우 고도의 인지적 서명이다.

**인터뷰 연계**: 궤변을 거부한 응시자에게 "이 정보를 왜 사용하지 않았습니까?"를 물으면, "논리는 맞아 보이는데 뭔가 이상했습니다"(직관적 탐지)와 "전제 A가 정보 B와 충돌합니다"(분석적 탐지)가 구분된다. 유형 D를 채택한 응시자에게 "이 정보의 논리가 틀렸다는 걸 알고 있었습니까?"를 물으면, "몰랐습니다"(우연)와 "알았지만 결론이 다른 데이터와 일치해서 결론만 가져갔습니다"(고도 판별)가 분리된다.

**현재 상태**: 프레임워크 내에서 측정 가능하나, 과제 설계의 핵심 난제가 미해결. 아래 9.5절 참조.

### 9.3 지식과 지혜의 분리

**개요**: 본 시스템은 마이크로월드의 자체 규칙 환경을 사용하므로, 현실 세계에서 축적한 지식이 직접적으로 유리하게 작용하지 않는다. 따라서 "지식이 많은 사람"과 "지식은 적지만 사고 과정이 뛰어난 사람"은 이미 어느 정도 분리된다.

**부분적으로 측정 가능한 지혜의 영역**: 메타인지(자기 한계 인식), 충동 억제(행동하지 않는 것이 최적인 상황 판별), 장기적 판단(즉각적 이득을 포기하고 장기적으로 유리한 선택), 인간관계 시나리오에서의 맥락적 판단. 이들은 지혜의 외적 표현이며 로그에서 포착 가능하다. 또한 인지적 몰입 의지(3.3절)도 지혜와 관련된다 — 필요한 깊이까지는 거부 없이 진입하되 불필요한 깊이에서는 멈추는 판단은 지혜의 표현이다.

**구조적으로 측정 불가능한 지혜의 영역**: 가치 판단이다. 현재 시스템은 "본인에게 이득이 되는 행동을 선택했는가"를 기준으로 평가한다. 그러나 지혜로운 판단은 때로 본인에게 불리한 선택이다. 자기 이득을 포기하면서 더 큰 맥락에서 올바른 판단을 내리는 능력은 현재 평가 기준에서 오히려 낮은 점수를 받을 수 있다. 또한 경험의 깊이에서 비롯된 맥락적 통찰은 마이크로월드의 제한된 규칙 안에서 드러나기 어렵다.

**잠재적 해결 방향**: 과제에 윤리적 딜레마를 내장하여 "무엇을 이득으로 정의하는가"를 응시자가 스스로 결정하게 하는 설계가 가능하다. 그러나 이 경우 채점 기준이 규범적(normative)이 되어 객관적 측정 시스템의 전제와 충돌한다.

**현재 상태**: 지식과 인지 능력의 분리는 달성됨. 지혜의 일부 영역은 기존 메트릭으로 포착 가능. 가치 판단 영역은 설계 원리 수준의 미해결 과제.

### 9.4 관점 전환 및 다중 시점 통합 (Perspective-taking & Multi-viewpoint Integration)

**개요**: 동일한 구조를 서로 다른 이해관계의 시점에서 각각 바라보고, 각 시점에서 합리적인 판단을 내릴 수 있는지, 나아가 여러 시점을 통합하여 전체 구조 안에서 판단을 내릴 수 있는지를 측정한다. 자기 입장에 고착되어 하나의 시점에서만 최적화하는 사람과, 복수의 시점을 동시에 운용하며 전체 구조를 파악하는 사람은 인지적으로 완전히 다르다. 이는 종합지능에서 빠지면 안 되는 차원이다.

**과제 설계**: 응시자에게 강제로 역할을 전환시키며 과제를 수행하게 한다. 1라운드는 역할 A(예: 자원 생산자)로서 최적 전략을 수립한다. 2라운드는 같은 환경에서 역할 B(예: 자원 배분자)로서 전략을 수립한다. 3라운드는 역할 C(예: 조율자)로서 양 집단을 조율한다.

**핵심 메트릭**:

- 관점 수(Perspective Count): 응시자가 몇 개의 역할에서 각각 합리적 전략을 구사할 수 있는가. 단일 역할에서만 유효한 전략을 세우는 사람과 모든 역할에서 유효한 전략을 세우는 사람의 차이.
- 관점 간 연결성(Cross-perspective Linkage): 1라운드에서 세운 전략을 2라운드에서 상대 입장으로 전환했을 때 예측하고 대응하는가, 아니면 이전 라운드의 자기 전략을 무시하고 독립적으로 행동하는가. 로그에서 이전 라운드와 동일한 정보를 재조회하되 다른 순서로 탐색하는 패턴이 관점 전환의 서명이다.
- 관점 통합(Perspective Integration): 3라운드(조율자)에서 1라운드와 2라운드 양쪽의 전략을 모두 고려한 조율안을 내놓는지, 아니면 한쪽에 편향된 결정을 내리는지. 양쪽 역할에서 접근한 정보를 균형 있게 재조회하는 패턴이 통합의 서명이다.

**소재의 추상화와 현실 맥락의 트레이드오프**: 이 유형의 소재로 현실의 정치적·사회적 구조(노동자-국가-자본가 등)를 직접 사용하면, 응시자의 기존 신념 체계가 개입하여 "신념을 초월하는 관점 전환 능력"까지 측정할 수 있다. 그러나 현실 소재를 사용하는 순간 시스템에 대한 정치적 편향 공격이 들어온다. 실제로 편향이 있든 없든, 편향이 있다는 인식 자체가 시스템의 외부 신뢰도를 훼손한다.

이를 완화하기 위해 두 가지 버전을 설계한다:

- 추상화 버전: "자원 생산자-조율자-자원 배분자" 같은 추상적 역할로 치환한다. 구조는 동일하지만 정치적 프레임이 제거된다. 정치적 안전성이 높으나, 기존 신념을 초월하는 능력은 측정하지 못한다.
- 현실 맥락 버전: 실제 사회적 구조를 소재로 사용한다. 측정의 깊이는 높으나 정치적 공격 표면이 넓다.

운영 환경의 정치적 민감도에 따라 어떤 버전을 배포할지를 선택하는 구조로 운영한다. 채점 기준은 양 버전 모두 동일하게 "어떤 입장을 취했는가"가 아니라 "몇 개의 입장을 구사할 수 있었는가, 그리고 입장 간 통합이 이루어졌는가"이다. 이러면 어떤 정치적 입장에도 유불리가 없다.

**현재 상태**: 추상화 버전은 기존 프레임워크 내에서 과제 설계로 구현 가능. 현실 맥락 버전은 정치적 안전성과 측정 깊이의 트레이드오프가 미해결. 아래 9.5절 참조.

### 9.5 미해결 설계 난제

본 절은 위 보완 과제들을 구현하기 위해 선행적으로 해결해야 하는 설계 난제들을 정리한다.

**규칙 모호성과 채점 안정성의 균형**: 궤변(9.2절 유형 C)을 마이크로월드 안에서 성립시키려면, 규칙 자체에 다층적 해석 가능성이 내장되어야 한다. 규칙이 완전히 명확하면 궤변이 성립할 수 없고, 규칙이 모호하면 채점 기준이 불안정해진다. "충분히 모호해서 궤변이 가능하지만, 충분히 명확해서 정답/오답 판정이 가능한" 규칙 설계가 필요하다. 이 균형점을 찾는 것이 궤변 탐지 과제의 핵심 선행 과제이다.

**"이득"의 다층적 정의**: 지혜 측정(9.3절)을 위해 과제에 윤리적 딜레마를 도입하면, 어떤 선택이 "이득"인지를 단일 기준으로 판정할 수 없게 된다. 단기 이득과 장기 이득, 개인 이득과 집단 이득, 물질적 이득과 관계적 이득이 충돌하는 상황에서 어떤 선택에 높은 점수를 줄 것인가는 평가 체계가 아니라 가치 체계의 문제이다. 객관적 측정 시스템이 가치 판단을 내포하는 순간, 시스템의 문화적 중립성(7.6절)이 훼손될 수 있다. 이를 해결하려면 "어떤 선택이 좋은가"가 아니라 "선택의 근거가 얼마나 체계적이고 일관되는가"를 평가 기준으로 전환하는 방법이 유력하나, 이 경우에도 체계적인 악의적 판단과 체계적인 이타적 판단을 동일 점수로 산출하는 것이 타당한가라는 후속 질문이 남는다.

**유형 D 정보의 과제 내 검증 경로 보장**: 논리가 전부 오류이지만 결론이 맞는 정보(9.2절 유형 D)를 마이크로월드에 배치할 때, 해당 결론을 독립적으로 검증할 수 있는 대체 경로가 과제 안에 존재해야 한다. 대체 경로가 없으면 유형 D를 폐기한 사람이 올바른 결론에 도달할 수단 자체가 사라지므로, 논리 검증 능력이 높은 사람이 구조적으로 불이익을 받게 된다. 대체 경로의 존재와 접근 난이도를 어떻게 설정할 것인가가 과제 설계 시 결정되어야 한다.

**함정 정보와 구조 변이의 상호작용**: 구조 변이(2.4절)로 매번 다른 규칙 세트가 배정되는 환경에서, 함정 정보와 궤변의 난이도가 구조 세트에 따라 체계적으로 달라질 수 있다. 특정 구조 세트에서는 궤변이 쉽게 탐지되고, 다른 세트에서는 극도로 어려울 수 있다. 이를 IRT 모델(5.3절)로 보정할 수 있지만, 궤변의 난이도를 사전에 정량화하는 방법론 자체가 아직 확립되지 않았다. 대규모 파일럿 테스트를 통한 경험적 난이도 산출이 선행되어야 한다.

**인간관계 시나리오에서의 궤변 자연성**: 정보 구조형 마이크로월드에서의 궤변은 데이터 간 모순으로 설계할 수 있으나, 인간관계 시나리오에서의 궤변은 대화 상대의 발화로 제시되어야 한다. 대화 상대가 논리적이지만 궤변적인 주장을 할 때 응시자가 이를 탐지하고, 거부하고, 대화를 어떻게 이어가는지를 측정하려면, 시나리오의 대화 분기 구조가 매우 정교해야 한다. 응시자의 반응에 따라 상대의 후속 발화가 달라져야 하므로, 동적 대화 트리의 설계 복잡도가 높다.

**관점 전환 과제의 정치적 안전성과 측정 깊이의 균형**: 관점 전환형 과제(9.4절)에서 추상화된 역할을 사용하면 정치적 공격은 줄지만, 응시자의 기존 신념 체계가 개입하지 않으므로 "신념을 초월하는 관점 전환"은 측정하지 못한다. 현실 맥락을 사용하면 측정 깊이는 높아지지만 "이 프레임 자체가 편향적이다"라는 비판이 가능하다. 어떤 수준의 추상화가 최적의 균형점인지에 대한 경험적 연구가 필요하다. 또한 추상화 버전과 현실 맥락 버전이 동일한 인지 차원을 동일한 정밀도로 측정하는지에 대한 등가성 검증(equivalence validation)이 선행되어야 한다.

**관점 간 연결성의 채점 기준 표준화**: 1라운드의 전략과 2라운드의 전략이 "연결되어 있다"는 판정을 로그만으로 내리기 위한 객관적 기준이 필요하다. 이전 라운드에서 접근한 정보를 다음 라운드에서 재조회하는 패턴이 관점 연결의 프록시가 될 수 있으나, 단순 재조회와 관점 전환적 재조회를 구분하는 기준이 모호하다. 인터뷰로 보완할 수 있으나, 라운드 수가 늘어나면 인터뷰의 질문 수도 비례하여 증가하므로 실행 부담이 커진다.

| 계단형 정보 구조 | 하위 정보의 조합으로만 상위 정보에 접근할 수 있는 계층적 정보 배치 방식 |
| 탐색 서명 | 피험자의 정보 조회 순서와 빈도가 형성하는 고유한 행동 패턴 |
| 전환 지연 | 연속된 두 행동 사이의 시간 간격으로, 내적 연산 복잡도의 프록시 |
| 일치-불일치 매트릭스 | 로그 기반 해석과 인터뷰 기반 해석 간의 일관성을 평가하는 분석 도구 |
| 구조 변이 | 과제의 논리적 연결 규칙 세트를 여러 벌 준비하여 무작위 배정하는 방식 |
| g-factor | 다양한 인지 과제에서 공통으로 추출되는 일반 지능 잠재 변인 |
| IRT | 문항 반응 이론. 문항 난이도와 피험자 능력치를 동시에 추정하는 통계 모델 |
| DIF | 차등 문항 기능. 동일 능력의 피험자가 특정 집단 소속에 따라 다른 응답 확률을 보이는지를 검증하는 분석 |
| Cohen's kappa | 두 채점자 간 일치도를 우연에 의한 일치를 보정하여 산출하는 지표 |
| 비언어적 확인 경로 | 언어적 재구성 대신 시각적 경로 제시 및 선택을 통해 사고 과정을 확인하는 인터뷰 대체 방식 |
| 복잡도 비례성 | 정보 체류 시간이 해당 정보의 복잡도에 비례하는지를 통해 실제 정보 처리 여부를 판별하는 기준 |
| 환경 변화 감소율 | 평상시 수행 속도 대비 환경 변화 직후 수행 속도의 감소 비율. 엘리트 수행자와 구조 학습자를 구분하는 핵심 지표 |
| 과제 배터리 | 서로 다른 인지 차원을 자극하는 다양한 소재의 과제 묶음. 단일 소재의 영역 특수적 능력이 아닌 종합적 인지 능력을 측정하기 위한 구조 |
| 직관 측정형 과제 | 논리적 풀이가 가능하나 주어진 시간 내에는 불가능한 복잡도를 갖는 과제. 과정 메트릭 대신 정확도-속도 조합으로 압축된 내적 사고를 추정 |
| 출력 채널 독립성 | 인지 측정이 특정 입력 장치나 신체 조건에 의존하지 않아야 한다는 설계 원칙. 물리적 조작 시간을 인지 처리 시간에서 분리하고, 다양한 입력 장치에서 동일한 로그 구조를 보장 |
| Fitts의 법칙 | 대상까지의 거리와 대상의 크기로 물리적 이동 시간을 예측하는 모델. 커서 이동 시간을 추정하여 인지 처리 시간에서 차감하는 데 사용 |
| 입력 장치별 보정 계수 | 시선 추적, 음성 입력, 스위치 입력, BCI 등 각 입력 장치의 물리적 조작 시간 분포를 별도 측정하여 산출하는 보정값. 장치 간 시간 메트릭의 등가성을 확보 |
| 함정 정보 | 계단형 정보 구조 내에 의도적으로 배치된 논리적 모순, 데이터 불일치, 또는 맥락적 부정합 정보. 비판적 사고 능력을 측정하기 위한 과제 설계 요소 |
| 인지적 오염도 | 함정 정보를 거부한 뒤에도 관련 정보를 반복 재조회하는 빈도. 거부가 인지적으로 완결되었는지 여부를 측정하는 지표 |
| 궤변 (유형 C 정보) | 논리의 형식은 완벽하지만 결론이 틀린 정보. 논리적 검증만으로는 탐지할 수 없으며, 전제 의심 또는 결론 독립 검증 능력이 필요 |
| 역설적 정합 (유형 D 정보) | 논리는 전부 오류이지만 결론이 맞는 정보. 논리와 결론을 분리하여 독립적으로 평가할 수 있는 고도의 인지 능력을 측정 |
| 거부 후 자원 배분 효율성 | 함정 정보를 파악한 뒤 해당 정보에 계속 시간을 소비하는지, 즉시 원래 탐색 경로로 복귀하는지의 차이를 나타내는 메트릭 |
| 관점 수 | 응시자가 서로 다른 역할에서 각각 합리적 전략을 구사할 수 있는 역할의 수. 단일 시점 고착과 다중 시점 운용을 구분하는 지표 |
| 관점 간 연결성 | 역할 전환 시 이전 역할의 전략을 현재 역할의 관점에서 예측하고 대응하는 능력. 라운드 간 정보 재조회 패턴으로 측정 |
| 관점 통합 | 복수의 역할에서 경험한 전략과 정보를 균형 있게 종합하여 전체 구조 안에서 판단을 내리는 능력 |
| 소재 추상화 | 정치적·사회적 맥락이 있는 소재를 추상적 역할로 치환하여 정치적 편향 공격 표면을 줄이는 과제 설계 기법. 측정 깊이와 트레이드오프 관계 |
| 능력 천장 | 다음 단계로 진입하기 위한 단서 자체를 발견하지 못하여 탐색이 멈추는 현상. 인지 능력의 상한을 반영 |
| 의지 천장 | 다음 단계의 단서를 발견했지만 깊이 들어가는 것을 거부하여 탐색이 멈추는 현상. 인지적 몰입 저항을 반영. 능력 천장과 로그 서명이 다르다 |
| 인지적 몰입 저항 | 깊은 사고로 진입하는 데 대한 심리적 거부감. 인지적 불편함 회피, 사회적 압력의 내면화, 불확실성에 대한 불안 등으로 발생 |
| 환경 변화 후 오류율 | 환경 변화 직후 행동에서 발생하는 오류의 비율. 반응 시간과 결합하여 충동적 행동과 숙련된 판단을 구분하는 지표 |
