# CPAS 설계명세서 — 제1절: 시스템 의도

> **Cognitive Process Assessment System (CPAS)**
> 본 문서는 CPAS 설계명세서 제1절 "시스템 의도"를 독립 문서로 분기하여 상세화한 것이다.

---

## 1.1 시스템의 존재 이유

CPAS는 **사람의 종합지능지수를 측정하는 평가 시스템**이다.

기존의 지능 측정 도구(IQ 테스트, 적성 검사, 설문 기반 평가)는 "정답을 맞혔느냐"라는 결과만을 본다. 그러나 동일한 정답에 도달하더라도 그 과정에는 질적으로 완전히 다른 사고 구조가 존재한다. 운 좋게 맞은 사람과 깊이 사고한 사람은 결과지에서 구분되지 않는다.

CPAS가 측정하려는 것은 **정답이 아니라, 정답에 도달하는 과정의 구조**다.

구체적으로:

- 생각을 얼마나 넓게 발산했는가
- 발산된 경우의 수 중에서 원하는 선택지를 얼마나 정확히 찾아냈는가
- 추론 체인이 몇 단계까지 내려갔는가 (사고의 깊이)
- 서로 다른 사고 분기들이 유기적으로 연결되어 있는가
- 상황이 변했을 때 얼마나 빠르게 재계획했는가
- 자기 사고 과정을 얼마나 정확히 인식하고 있는가

이것들은 설문이나 선다형 문항으로는 측정이 불가능하다. 설문은 응시자의 **선언적 자기보고**에 의존하고, 선다형은 탐색 과정을 기록하지 않는다.

---

## 1.2 핵심 철학: 생각의 그림자를 캐스팅하라

CPAS의 설계 철학은 하나의 명제로 요약된다:

> **"생각은 직접 관측할 수 없다. 따라서 생각이 외부로 드러나는 행동의 흔적을 강제로 만들어내는 환경을 설계해야 한다."**

이것을 **"사고의 그림자 캐스팅(Shadow Casting of Thought)"**이라 부른다.

기존 평가는 응시자에게 **선택지**를 준다 — A, B, C, D 중 골라라. 이 방식은 "어떤 순서로 생각했는지", "어떤 정보를 먼저 확인했는지", "되돌아가서 다시 봤는지"를 전혀 포착하지 못한다.

CPAS는 선택지가 아니라 **행동 공간**을 준다. 마이크로월드(Microworld)라 불리는 시뮬레이션 환경 안에서 응시자가 자유롭게 정보를 탐색하고, 행동하고, 전략을 수정하게 한다. 이 과정에서 발생하는 모든 행동은 1ms 단위로 로깅된다. 로그에 기록된 행동의 시퀀스, 타이밍, 패턴이 곧 **사고 구조의 외부화된 그림자**가 된다.

핵심 전제:

- **행동 공간이 충분히 넓어야 한다** — 선택지가 제한되면 사고의 개인차가 행동에 반영되지 않는다.
- **정보가 능동적으로 열어봐야 보이게 해야 한다** — "어떤 정보를 어떤 순서로 열어봤는가"가 곧 사고의 탐색 트리를 외부화한 것이다.
- **깊이 있게 생각하지 않으면 절대 접근하지 않을 정보가 존재해야 한다** — 이것이 사고 깊이의 프록시(proxy)가 된다.

## 1.3 기존 방식이 실패하는 이유

CPAS가 존재해야 하는 이유를 명확히 하기 위해, 기존 측정 방식의 구조적 한계를 정리한다.

### 1.3.1 IQ 테스트 / 선다형 적성 검사

| 문제 | 설명 |
|------|------|
| 결과만 측정 | 정답 여부만 기록하며, 도달 과정의 질을 구분하지 못한다 |
| 사고 과정 비가시 | "어떤 순서로 생각했는가"를 포착할 채널이 없다 |
| 운과 실력의 미분리 | 우연히 맞은 경우와 체계적으로 추론한 경우를 동일하게 취급한다 |
| 시간 축 부재 | 사고의 시간적 전개(발산 → 수렴 사이클)를 기록하지 않는다 |
| 고정된 난이도 | 응시자의 능력에 따라 과제 복잡도가 조절되지 않는다 |

### 1.3.2 설문 / 자기보고 기반 평가

| 문제 | 설명 |
|------|------|
| 선언적 자기보고 의존 | 응시자가 실제로 어떻게 사고하는지가 아니라, 자신이 어떻게 사고한다고 *믿는지*를 측정한다 |
| 사회적 바람직성 편향 | 응시자가 "좋아 보이는 답"을 고를 동기가 있다 |
| 메타인지 능력 혼입 | 자기보고의 정확성 자체가 메타인지 능력에 의존하므로, 측정 대상과 측정 도구가 순환 관계에 빠진다 |
| 행동 관측 부재 | 실제 상황에서의 의사결정 패턴을 전혀 반영하지 않는다 |

### 1.3.3 면접 / 관찰 기반 평가

| 문제 | 설명 |
|------|------|
| 평가자 주관 | 채점자의 인상, 편향, 피로도에 따라 점수가 변동한다 |
| 표준화 불가 | 면접 상황이 매번 다르므로, 응시자 간 공정 비교가 어렵다 |
| 확장성 부재 | 대규모 집단에 적용할 수 없다 |

CPAS는 이 세 범주의 한계를 모두 우회하는 네 번째 범주를 정의한다: **행동 로그 기반 과정 측정(Process-Based Assessment via Behavioral Logging)**.

## 1.4 측정 대상: 9개 인지 차원

CPAS는 단일 점수를 산출하지만, 그 이면에는 9개의 독립적인 인지 차원에 대한 측정이 존재한다. 이 차원들은 "지능이란 무엇인가"에 대한 CPAS의 조작적 정의이기도 하다.

### 차원 1: 발산적 사고 (Divergent Thinking)

경우의 수 탐색의 **폭**과 **체계성**.

단순히 많은 가능성을 떠올리는 것이 아니라, 탐색 공간을 체계적으로 커버하는 능력을 의미한다. 무작위로 많이 탐색하는 것과, 구조적으로 빠짐없이 탐색하는 것은 질적으로 다르다.

측정 기준:
- 탐색한 경우의 수의 총량
- 탐색 순서의 체계성 (무작위 vs 구조적 순회)
- 탐색 공간의 커버리지 (놓친 영역이 있는가)
- 중복 탐색 비율 (같은 경로를 반복하는가)

### 차원 2: 수렴적 사고 (Convergent Thinking)

발산된 경우의 수 중에서 **최적 선택지를 찾아내는 정확도**.

많은 가능성을 열어놓은 뒤, 그 중 상황에 가장 적합한 하나를 골라내는 능력이다. 발산 없이 수렴만 하면 시야가 좁고, 발산만 하고 수렴하지 못하면 결정을 내리지 못한다.

측정 기준:
- 최종 선택의 객관적 최적성
- 발산 → 수렴 전환 지점의 타이밍 적절성
- 선택에 도달하기까지 소거한 대안의 수와 소거 근거의 합리성

### 차원 3: 사고의 깊이 (Reasoning Depth)

추론 체인의 **단계 수**.

"A이면 B, B이면 C, C이면 D"처럼 연쇄 추론이 몇 단계까지 이어지는가를 측정한다. 1~2단계 추론만 하는 사람과 5~6단계까지 내려가는 사람은 동일한 문제에서 완전히 다른 행동 패턴을 보인다.

측정 기준:
- 과제 내에서 도달한 추론 체인의 최대 단계 수
- 깊은 추론이 필요한 정보에 접근했는가 (얕은 사고로는 절대 열어보지 않을 정보)
- 추론 깊이와 정확도의 관계 (깊이 들어갈수록 오류가 증가하는가, 유지되는가)

### 차원 4: 분기 간 연관성 (Inter-Branch Connectivity)

서로 다른 사고 분기들의 **유기적 연결**.

발산적 사고가 여러 갈래로 뻗어나갈 때, 각 갈래가 독립적으로 존재하는 것이 아니라 서로 정보를 주고받으며 영향을 미치는지를 측정한다. 한 분기에서 얻은 통찰이 다른 분기의 탐색 방향을 바꾸는 것이 높은 연관성의 표지다.

측정 기준:
- 분기 A의 탐색 결과가 분기 B의 탐색 전략에 영향을 미친 횟수
- 서로 다른 분기의 정보를 통합하여 새로운 추론을 도출한 빈도
- 분기 간 전환 패턴의 규칙성 (무작위 전환 vs 전략적 전환)

### 차원 5: 적응적 재계획 (Adaptive Replanning)

상황 변화에 대한 **대응 속도와 질**.

환경이 변했을 때 기존 계획을 얼마나 빠르게 폐기하고 새로운 계획을 수립하는가. 단순히 빠르기만 한 것이 아니라, 새 계획의 질이 기존 계획과 비교하여 상황에 더 적합한지가 관건이다.

측정 기준:
- 환경 변화 후 첫 행동까지의 지연 시간
- 기존 계획에 대한 집착도 (변화 후에도 무효화된 전략을 고수하는가)
- 새 계획의 질 (변화된 상황에 대한 적합성)
- 변화 탐지 자체의 속도 (변화가 일어났다는 것을 얼마나 빨리 알아챘는가)

### 차원 6: 메타인지 (Metacognition)

**자기 사고 과정에 대한 인식의 정확도**.

자신이 무엇을 알고 무엇을 모르는지, 자신의 판단에 얼마나 확신을 가져야 하는지, 현재 전략이 효과적인지 아닌지를 정확히 아는 능력이다.

이 차원은 과제 수행 로그만으로는 완전히 포착하기 어려우며, 인터뷰 보정(제4절)이 필수적으로 개입하는 유일한 차원이다. 과제 후 인터뷰에서 "당신은 이 시점에서 이런 행동을 했는데, 왜 그랬습니까?"라는 질문에 대한 답변의 정확도가 곧 메타인지 수준의 프록시가 된다.

측정 기준:
- 로그 기반 행동과 사후 자기 보고의 일치도
- 확신도 보정(calibration): 자신이 "확실하다"고 한 판단의 실제 정확도
- 전략 전환 시점의 자각 ("전략을 바꿨다"는 것을 인식했는가)
- 자기 한계 인식 ("이건 모르겠다"고 정확히 판단한 빈도)

### 차원 7: 관점 전환 (Perspective Shifting)

**다중 시점의 운용 및 통합 능력**.

하나의 상황을 여러 입장에서 바라보고, 각 입장에서의 판단을 수행한 뒤, 그것들을 하나의 통합된 이해로 종합하는 능력이다. 단순히 "상대방 입장에서 생각해봐"가 아니라, 복수의 시점에서 독립적으로 사고한 뒤 그 결과물들 사이의 관계를 파악하는 것까지를 포함한다.

측정 기준:
- 동일 상황에서 서로 다른 역할/입장을 수행할 때의 행동 차이의 적절성
- 관점 전환 후 이전 관점의 추론을 활용하는 빈도
- 다중 관점을 통합한 최종 판단의 질
- 자기 입장과 타인 입장의 분리 정확도

### 차원 8: 비판적 사고 (Critical Thinking)

**모순, 궤변, 허위 정보의 탐지 능력**.

주어진 정보를 그대로 수용하지 않고, 내적 일관성을 검증하고, 논리적 오류를 식별하며, 의도적으로 삽입된 함정 정보를 걸러내는 능력이다.

이 차원은 보완 과제(제9절)에서 별도의 전용 과제를 통해 측정된다: 함정 정보 탐지 과제, 궤변 탐지 과제(유형 A~D).

측정 기준:
- 과제 내 의도적으로 삽입된 모순의 탐지율
- 논리적으로 유효하지만 사실적으로 거짓인 정보(유형 C)의 식별
- 사실적으로 유효하지만 논리적으로 무효인 주장(유형 B)의 식별
- 정보의 출처와 맥락을 고려한 신뢰도 평가 능력

### 차원 9: 인지적 몰입 의지 (Cognitive Engagement Will)

**필요한 깊이까지 도달하는 저항 없는 진입과 적절한 중단**.

이 차원은 "능력"과 "의지"를 분리하기 위해 존재한다. 어떤 사람은 깊이 사고할 능력이 있지만 그 깊이까지 들어가려는 의지가 부족하고, 어떤 사람은 의지는 있지만 능력이 따라가지 못한다. CPAS는 이 둘을 구분한다.

이것은 곧 **능력 천장(Ability Ceiling)**과 **의지 천장(Will Ceiling)**의 분리로 이어진다(제3절에서 상세화).

측정 기준:
- 과제 난이도가 증가할 때 포기 시점 vs 실패 시점의 구분
- 어려운 구간에서의 탐색 지속 시간
- 보상이 불분명한 상태에서의 탐색 유지 여부
- 인지적 부하가 높아질 때의 행동 패턴 변화 (전략적 후퇴 vs 무기력한 포기)

## 1.5 출력 채널 독립성 원칙

CPAS의 설계에서 가장 중요한 제약 조건 중 하나:

> **"인지 측정은 출력 채널에 독립적이어야 한다."**

이것이 의미하는 바:

키보드, 마우스, 시선 추적, 음성 입력, 터치스크린, 뇌-컴퓨터 인터페이스(BCI) 등 어떤 입력 장치를 사용하든, 로그에 기록되는 **인지적 선택의 구조**는 동일해야 한다. 물리적 조작에 소요되는 시간(운동 시간)은 인지 처리 시간에서 분리되어야 한다.

이 원칙이 필요한 이유:

- 신체 장애가 있는 응시자가 동일한 인지 능력을 가졌음에도 불리한 평가를 받아서는 안 된다.
- 타이핑 속도나 마우스 조작 숙련도가 인지 능력 점수에 혼입되어서는 안 된다.
- 미래에 새로운 입력 장치가 등장하더라도 평가 프레임워크를 재설계할 필요가 없어야 한다.

구현 방법:

- 모든 행동 로그에서 **"무엇을 했는가"(인지적 선택)**와 **"얼마나 걸렸는가"(물리적 실행 시간)**를 분리하여 기록한다.
- 인지적 선택의 시퀀스와 전략적 패턴은 채점에 사용하되, 물리적 실행 시간은 채점에서 제외한다.
- 단, 인지 처리 시간과 물리적 실행 시간을 완벽히 분리하는 것이 기술적으로 불가능한 경우, 입력 장치별 보정 계수를 적용한다.
- 시선 추적이나 BCI를 통해 순수 인지 처리 시간에 더 가까운 측정이 가능해지면, 이를 기준 채널로 사용하여 다른 입력 장치의 보정 계수를 교정한다.

## 1.6 "이득"의 정의

CPAS의 평가 기본 전제에는 다음 문장이 포함된다:

> "제한 시간 안에 관측된 행동이 **본인에게 이득이 되느냐** 아니냐"

여기서 "이득"은 단순히 점수를 많이 받는 것이 아니다. CPAS에서 이득의 정의는 다층적이다:

- **1차 이득 (과제 내 목표 달성):** 마이크로월드 안에서 주어진 목표를 달성했는가.
- **2차 이득 (차선 대비 우위):** 같은 상황에서 가능했던 다른 행동 대비 더 나은 결과를 가져왔는가.
- **3차 이득 (미래 대응 준비):** 현재 행동이 다음 상황에 대한 대응 가능성을 높였는가. 즉, 단기 이득을 포기하고 장기 이득을 선택하는 전략적 판단도 "이득"에 포함된다.

이 다층적 정의는 "이득"을 주관적 가치 판단으로 만들 위험이 있으며, 이는 미해결 설계 난제 중 하나로 9.5절에 등재되어 있다. 현재 CPAS의 운영 원칙은 다음과 같다:

- **과제 설계 시 "이득"의 기준을 객관적으로 정의 가능한 수준으로 한정한다.**
- 모호한 가치 판단이 개입되는 과제에서는 "이득"을 기준으로 채점하지 않고, 대신 사고 과정의 구조적 특성(깊이, 폭, 연관성)만을 채점한다.

## 1.7 측정 철학의 요약

CPAS의 시스템 의도를 관통하는 철학을 5개 명제로 정리한다:

**명제 1: 과정이 결과보다 더 많은 정보를 담고 있다.**
동일한 결과에 도달하는 과정은 무한히 다양하며, 과정의 질이 곧 인지 능력의 차이다. 결과만 측정하는 시스템은 정보의 대부분을 버리고 있다.

**명제 2: 생각은 관측 불가능하지만, 행동 공간이 충분히 넓으면 사고의 구조가 행동에 투영된다.**
선택지가 A/B/C/D 뿐이면 사고 과정이 숨겨지지만, 자유도 높은 행동 공간에서는 개인의 인지 전략이 행동 패턴으로 드러난다. 이것이 마이크로월드 기반 설계의 근거다.

**명제 3: 시간 축이 인지의 차원을 분리한다.**
즉각(0~3초), 단기(10~60초), 중기(3~10분), 장기(30분~수 시간)에서 작동하는 인지 기제는 서로 다르다. 동일 인물이라도 시간 스케일에 따라 인지 능력 프로필이 달라질 수 있으며, CPAS는 이를 분리하여 측정한다.

**명제 4: 능력과 의지는 분리되어야 한다.**
"할 수 있는데 하지 않는 것"과 "하고 싶은데 할 수 없는 것"은 완전히 다른 현상이다. 기존 평가 시스템은 이 둘을 하나의 점수에 섞어버린다. CPAS는 능력 천장과 의지 천장을 구분하여, 응시자의 인지적 잠재력과 실행 의지를 별개의 차원으로 평가한다.

**명제 5: 측정은 신체를 통과하되 신체에 의존하지 않아야 한다.**
모든 인지적 선택은 결국 신체를 통해 외부로 표현되지만, 신체적 조건(반응 속도, 운동 능력, 감각 장애)이 인지 능력 점수를 오염시켜서는 안 된다. 이것이 출력 채널 독립성 원칙이다.

## 1.8 CPAS가 아닌 것

시스템의 의도를 명확히 하기 위해, CPAS가 **명시적으로 측정하지 않는 것**을 정의한다.

| CPAS가 아닌 것 | 이유 |
|----------------|------|
| 지식 테스트 | CPAS는 "무엇을 아느냐"가 아니라 "어떻게 생각하느냐"를 측정한다. 지식의 양은 채점 대상이 아니다. |
| 성격 검사 | 외향성, 개방성 등 성격 특성은 CPAS의 측정 범위 밖이다. |
| 학업 성취도 평가 | 특정 교과의 학습 수준을 측정하지 않는다. |
| 창의성 테스트 | 발산적 사고를 측정하지만, "참신한 아이디어를 떠올리는가"는 측정하지 않는다. 독창성이 아닌 체계성을 본다. |
| 정서 지능 측정 | 감정 인식, 공감 능력 등은 별도의 도구가 필요하다. 관점 전환 차원이 일부 겹치지만, 정서적 공감은 CPAS의 범위 밖이다. |
| 도덕성 판단 | "올바른 선택을 했는가"는 채점하지 않는다. "어떤 구조로 판단에 도달했는가"만 채점한다. |

## 1.9 수능급 신뢰도라는 목표

CPAS가 목표로 하는 신뢰도 수준은 **대한민국 수능(College Scholastic Ability Test)에 준하는 수준**이다.

수능이 신뢰도를 확보하는 세 가지 기반:

1. **표준화된 환경:** 모든 응시자가 동일한 조건에서 시험을 본다.
2. **대규모 규준 집단:** 수십만 명의 데이터로 점수 분포를 안정화한다.
3. **문항 반응 이론(IRT) 기반 난이도 보정:** 개별 문항의 난이도와 응시자의 능력을 동시에 추정한다.

CPAS에 대응되는 설계:

1. **표준화:** 마이크로월드의 규칙 세트를 고정하되, 초기 조건(변수 값, 배치)을 무작위화하여 "같은 구조, 다른 표면"의 과제를 무한히 생성한다. 이것을 **구조 변이(Structural Variant)**라 부른다.
2. **규준 집단:** 연령대별, 교육 수준별 규준을 구축한다. 과제 소재 유형 중 "교육 커리큘럼 기반형"이 이 목적을 위해 존재한다.
3. **IRT 적용:** 각 프로세스 메트릭에 대해 IRT 모델을 적용하여, 과제의 난이도와 응시자의 능력을 동시에 추정한다.

목표 재검사 신뢰도(test-retest reliability): **r ≥ 0.85**

이 수치를 달성하기 위해 동일 응시자에게 구조적으로 동등하지만 표면이 다른 과제를 반복 실시하여 메트릭의 안정성을 검증한다.

## 1.10 종합 점수의 의미

CPAS의 최종 산출물은 **단일 종합 점수**다.

9개 차원의 개별 점수는 내부적으로 존재하지만, 응시자에게 전달되는 것은 하나의 숫자다. 이 설계 결정의 근거:

- 다차원 프로필을 전달하면 해석이 개인마다 달라져, 점수의 사회적 기능(선발, 배치, 비교)이 약화된다.
- 종합 점수의 가중치는 **설계자가 정하지 않는다.** 규준 집단 데이터의 공분산 구조에서 통계적으로 도출한다. 이는 g-factor(일반 지능 인자) 추출과 유사한 접근이다.
- 가중치가 데이터에서 도출되므로, 규준 집단이 바뀌면 가중치도 자연스럽게 재조정된다.

단, 종합 점수가 "이 사람은 정확히 이런 사람이다"를 말해주지는 않는다. 종합 점수가 예측하는 것은 **"미지의 인지적 도전 과제에 직면했을 때, 이 사람이 효과적으로 대응할 확률"**이다. 이것이 CPAS가 "종합지능지수"라는 이름을 사용하는 이유다.

## 1.11 설계 전제 요약

본 절의 내용을 관통하는 설계 전제를 정리한다:

1. **1ms가 1틱 레이트를 완벽하게 보장한다는 가정** 하에 설계되었다.
2. 생각은 직접 관측할 수 없으므로, **행동 로그가 사고의 유일한 관측 채널**이다.
3. 행동 로그의 해상도가 사고 구조 추론의 정밀도를 결정한다. 따라서 **가능한 한 높은 시간 해상도(1ms)**로 기록한다.
4. **선택지를 주지 않고 행동 공간을 준다.** 이것이 사고의 개인차를 외부화하는 전제 조건이다.
5. 인지 측정은 **출력 채널(입력 장치)에 독립적**이어야 한다.
6. **능력과 의지는 구분되어야 한다.** 동일한 결과적 실패라도 원인이 다르면 다른 점수를 받아야 한다.
7. 종합 점수의 가중치는 **데이터에서 도출**되며, 설계자의 주관적 판단에 의존하지 않는다.
8. 과제 구조를 암기해도 재응시에서 이점을 얻을 수 없도록 **구조 변이와 이상 탐지**를 적용한다.
