# CPAS 설계명세서 — 제6절: 최종 평가

> **Cognitive Process Assessment System (CPAS)**
> 본 문서는 CPAS 설계명세서 제6절 "최종 평가"를 독립 문서로 분기하여 상세화한 것이다.

---

## 6.1 종합 점수: 단일 산출물

### 6.1.1 왜 단일 점수인가

CPAS의 최종 산출물은 **단일 종합 점수**다. 9개 인지 차원의 개별 점수는 내부 분석에 사용되지만, 응시자에게 전달되는 것은 하나의 숫자다.

단일 점수 결정의 근거:

- **해석의 통일성:** 다차원 프로필을 전달하면 해석이 개인마다 달라진다. "발산적 사고는 높은데 수렴적 사고는 낮은 사람"이 "발산적 사고는 낮은데 수렴적 사고는 높은 사람"보다 나은지 못한지를 판단하는 기준이 합의되지 않는다.
- **사회적 기능:** 선발, 배치, 비교 등의 목적에서 단일 점수가 다차원 프로필보다 실용적이다.
- **가중치의 객관성:** 다차원 프로필을 전달하면 사용자(고용주, 학교 등)가 자의적으로 가중치를 부여할 위험이 있다. CPAS는 데이터에서 도출된 가중치를 사용함으로써 이 자의성을 제거한다.

### 6.1.2 점수 체계

종합 점수는 규준 집단에 대한 상대적 위치로 표현된다. 구체적인 점수 체계(예: 평균 100, 표준편차 15 vs 평균 500, 표준편차 100)는 사회적 합의에 따라 결정되며, 본 명세서에서는 **통계적 구조**만 정의한다.

- 점수 분포는 규준 집단에서 정규분포를 따르도록 변환
- 점수에는 신뢰 구간을 첨부 (예: "당신의 점수는 X이며, 95% 신뢰 구간은 [X−a, X+b]입니다")
- 점수의 백분위 환산이 함께 제공됨

### 6.1.3 개별 차원 점수의 비공개

응시자에게는 종합 점수만 전달하고, 9개 차원의 개별 점수는 전달하지 않는다.

이유:
- 개별 차원 점수의 해석은 전문적 맥락 없이 오해를 유발할 수 있다
- "메타인지 점수가 낮다"는 정보가 자기충족적 예언으로 작용할 위험
- 개별 차원 점수를 조합하여 종합 점수를 역산하려는 시도를 방지
- 필요한 경우(임상적, 교육적 맥락) 공인된 전문가를 통해 개별 프로필을 열람할 수 있는 경로는 별도로 마련

## 6.2 점수의 예측적 의미

### 6.2.1 CPAS 점수가 예측하는 것

종합 점수가 예측하는 것은:

> **"미지의 인지적 도전 과제에 직면했을 때, 이 사람이 효과적으로 대응할 확률"**

이것이 "종합지능지수"라는 이름의 근거다. CPAS가 측정하는 것은 특정 영역의 전문성이 아니라, 영역에 무관한 일반적 인지 과정의 질이다.

### 6.2.2 CPAS 점수가 예측하지 않는 것

| 예측하지 않는 것 | 이유 |
|------------------|------|
| 특정 직업에서의 성공 | 직업적 성공은 인지 능력 외에 사회적 기술, 동기, 기회 등 다수의 요인에 의존 |
| 학업 성적 | 학업 성적은 지식 축적, 학습 환경, 동기 등 비인지적 요인이 큰 비중을 차지 |
| 창의적 성취 | 독창성은 CPAS의 측정 범위 밖 (발산적 사고의 체계성을 측정하지, 독창성을 측정하지 않음) |
| 도덕적 판단의 질 | CPAS는 "올바른 선택"을 채점하지 않음 |
| 행복이나 삶의 만족도 | 인지 능력과 주관적 안녕 사이에 직접적 인과 관계 없음 |

### 6.2.3 점수의 안정성

CPAS 점수는 단기적으로(수 주~수 개월) 안정적이어야 하며(재검사 신뢰도 r ≥ 0.85), 장기적으로는(수 년) 발달적 변화를 반영하여 변할 수 있다.

- 아동·청소년기: 발달에 따른 점수 상승이 정상
- 성인기: 대체로 안정적이되, 교육, 훈련, 경험에 의한 변화 가능
- 노년기: 일부 차원(특히 적응적 재계획, 작업기억 관련 지표)에서 감소 가능

## 6.3 응시자 경험 설계

### 6.3.1 설계 원칙

응시자의 경험은 다음 원칙을 따른다:

- **과제 자체가 흥미로워야 한다.** 마이크로월드는 "시험을 보고 있다"는 느낌보다 "흥미로운 문제를 풀고 있다"는 느낌을 주도록 설계한다. 응시자의 내적 동기를 활용해야 인지적 몰입 의지(차원 9)를 더 순수하게 측정할 수 있다.
- **불필요한 스트레스를 유발하지 않는다.** 시간 압박은 과제 설계의 일부로 존재하되, 시험 환경 자체에서 오는 불안은 최소화한다.
- **평가 과정이 투명해야 한다.** "무엇을 측정하는가"(인지 과정의 구조)는 사전에 설명하되, "어떻게 측정하는가"(구체적 메트릭, 채점 기준)는 공개하지 않는다.
- **평가가 끝난 후 학습 기회를 제공한다.** 응시자가 원하면, 점수 공개 후 자신의 행동 로그를 시각화하여 탐색 패턴, 전략 전환 등을 스스로 확인할 수 있는 피드백 세션을 제공한다.

### 6.3.2 시험 환경

- 통제된 시험장에서 진행 (원격 시험은 로그 무결성과 환경 통제 문제로 초기에는 지원하지 않음)
- 시험장 내 소음, 조명, 온도를 표준화
- 입력 장치는 응시자가 선택 가능 (키보드/마우스, 터치스크린 등)
- 캘리브레이션 과제를 통해 입력 장치별 보정 계수를 사전 측정

### 6.3.3 시간 구성

| 구간 | 소요 시간 | 내용 |
|------|-----------|------|
| 오리엔테이션 | 15~20분 | 시스템 설명, 입력 장치 선택, 캘리브레이션 |
| 과제 배터리 | 3~5시간 | 7개 소재 유형 × 4개 시간 스케일의 유효 조합 |
| 휴식 | 2~3회 × 10~15분 | 과제 사이에 배치 |
| 인터뷰 | 15~30분 | 로그 기반 구조화 인터뷰 |
| 총 소요 시간 | 4~6시간 | 오리엔테이션 + 과제 + 휴식 + 인터뷰 |

### 6.3.4 결과 통보

- 점수는 시험 후 **1~2주** 이내에 통보
- 로그 분석 파이프라인의 자동 처리 부분은 수 시간 내 완료되나, 인터뷰 보정(이중 채점)에 시간이 소요
- 결과 통보 시 포함 항목: 종합 점수, 신뢰 구간, 백분위, 규준 집단 정보
- 선택적 피드백 세션: 행동 로그 시각화와 함께 자기 인지 패턴에 대한 설명 제공

### 6.3.5 재응시 정책

- 최소 **6개월** 간격으로 재응시 가능
- 재응시 시 구조 변이에 의해 표면이 다른 과제 인스턴스를 받음
- 재응시 점수가 이전 점수와 크게 다른 경우 이상 탐지 프로토콜 작동 (8절)

---

## 6.4 점수의 사회적 사용에 대한 제한

CPAS는 점수의 생성과 전달까지만 책임지며, 점수의 사회적 사용(채용, 입학, 배치 등)에 대한 판단은 사용자(기관, 조직)의 책임이다. 다만, CPAS는 다음을 명시적으로 권고한다:

- CPAS 점수를 유일한 선발 기준으로 사용하지 말 것
- CPAS 점수의 신뢰 구간을 고려하여 경계선상의 응시자에 대해 추가 평가를 실시할 것
- CPAS 점수가 측정하지 않는 것(지식, 성격, 동기, 경험)을 별도로 평가할 것
- 특정 집단(신경다양성, 신체 장애 등)에 대한 체계적 편향 가능성을 인식할 것 (7절)
