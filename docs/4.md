# CPAS 설계명세서 — 제4절: 인터뷰

> **Cognitive Process Assessment System (CPAS)**
> 본 문서는 CPAS 설계명세서 제4절 "인터뷰"를 독립 문서로 분기하여 상세화한 것이다.

---

## 4.1 인터뷰의 역할 정의

### 4.1.1 보조 도구로서의 인터뷰

CPAS에서 인터뷰는 **주(主) 평가 도구가 아니라 보조 도구**다. 주 평가 도구는 마이크로월드에서의 행동 로그이며, 인터뷰는 로그만으로는 포착할 수 없는 정보를 보완하기 위해 존재한다.

인터뷰가 보조 도구인 이유:
- 인터뷰는 자기보고에 의존하므로, 선언적 편향의 위험이 있다.
- 인터뷰 능력(언어 표현력, 사회적 기술)이 인지 능력과 혼입될 수 있다.
- 채점자의 주관이 개입할 여지가 있다.

그럼에도 인터뷰가 필요한 이유:
- **메타인지(차원 6)**는 "자기 사고 과정에 대한 인식의 정확도"를 측정해야 하는데, 인식의 정확도를 평가하려면 "자기 인식"의 내용을 수집해야 한다. 행동 로그는 "무엇을 했는가"를 기록하지만, "본인이 무엇을 했다고 생각하는가"는 기록하지 못한다.
- **의지 천장과 능력 천장의 분리**(제3절)에서 경계 사례를 판별할 때, 응시자의 주관적 보고가 추가 정보를 제공한다.
- **직관 측정형 과제**(유형 5)에서 "왜 그렇게 판단했는가"에 대한 자기보고가 직관의 질을 간접적으로 평가하는 데 사용된다.

### 4.1.2 인터뷰가 절대로 해서는 안 되는 것

- 로그 데이터를 무시하고 인터뷰 응답만으로 채점하는 것
- 인터뷰에서 좋은 인상을 준 응시자에게 프로세스 메트릭 점수를 상향 조정하는 것
- 언어 표현 능력을 인지 능력으로 혼동하는 것
- 인터뷰어의 개인적 인상이 채점에 영향을 미치는 것

## 4.2 로그 기반 구조화 인터뷰

### 4.2.1 구조화의 의미

CPAS의 인터뷰는 **로그 기반 구조화 인터뷰(Log-Based Structured Interview)**다. 이것은 두 가지를 의미한다:

- **로그 기반:** 질문의 내용이 응시자의 실제 행동 로그에서 생성된다. 일반적 질문("당신은 문제를 어떻게 풀었습니까?")이 아니라, 구체적 행동에 기반한 질문("당신은 3분 42초 시점에서 변수 C를 확인한 직후 변수 F를 확인했습니다. 왜 그랬습니까?")이다.
- **구조화:** 질문의 유형, 순서, 채점 기준이 사전에 정의되어 있다. 인터뷰어의 재량에 따라 질문이 달라지지 않는다.

### 4.2.2 질문 생성 파이프라인

```
[행동 로그] → [자동 분석] → [주목할 행동 포인트 추출] → [질문 템플릿 매칭] → [구체적 질문 생성]
```

주목할 행동 포인트의 예:
- 전환 지연이 비정상적으로 긴 구간 → "이 시점에서 무엇을 생각하고 있었습니까?"
- 전략적 방향이 급격히 바뀐 시점 → "이 시점에서 왜 접근 방법을 바꿨습니까?"
- 깊은 정보 층위에 접근한 시점 → "이 정보를 확인하게 된 이유는 무엇입니까?"
- 높은 계단에서 포기한 시점 → "이 시점에서 왜 중단했습니까?"
- 오류 후 교정 행동을 취한/취하지 않은 시점 → "이 결과를 확인한 후 어떻게 판단했습니까?"

## 4.3 질문 유형 5종

### 유형 1: 행동 회고 질문 (Action Recall)

"당신은 [시점]에서 [행동]을 했습니다. 그때 무엇을 생각하고 있었습니까?"

**측정 대상:** 메타인지 (자기 행동에 대한 인식 정확도)

채점 기준:
- 실제 행동과 보고의 일치도
- 행동의 이유에 대한 설명의 논리적 일관성
- "기억나지 않는다"는 솔직한 답변은 감점하지 않음 (정확한 자기 인식)
- 실제 행동과 모순되는 사후 합리화는 메타인지 정확도를 낮추는 증거

### 유형 2: 전략 설명 질문 (Strategy Explanation)

"이 과제에서 당신의 전체적인 접근 방법은 무엇이었습니까?"

**측정 대상:** 메타인지 (전략 수준의 자기 인식)

채점 기준:
- 설명된 전략과 실제 행동 패턴의 일치도
- 전략의 추상화 수준 (구체적 행동 나열 vs 원칙 수준 설명)
- 전략 전환이 있었다면 전환 이유와 시점의 정확한 인식 여부

### 유형 3: 확신도 질문 (Confidence Calibration)

"이 결정에 대해 얼마나 확신합니까? (0~100%)"

**측정 대상:** 메타인지 (확신도 보정)

채점 기준:
- 보고된 확신도와 실제 정확도의 상관
- 확신도-정확도 캘리브레이션 곡선의 기울기와 절편
- 과신(overconfidence)과 과소 확신(underconfidence)의 방향과 크기

### 유형 4: 대안 탐색 질문 (Alternative Exploration)

"이 시점에서 다른 선택지는 무엇이 있었다고 생각합니까?"

**측정 대상:** 발산적 사고의 자기 인식, 수렴적 사고의 근거

채점 기준:
- 응시자가 인식한 대안의 수와 실제 존재했던 대안의 수의 비교
- 실제로는 탐색했지만 인식하지 못한 대안이 있는지
- 실제로는 탐색하지 않았지만 인식한 대안이 있는지 (사후 재구성)

### 유형 5: 한계 인식 질문 (Limitation Awareness)

"이 과제에서 가장 어려웠던 부분은 무엇입니까? 해결하지 못한 것이 있다면 무엇입니까?"

**측정 대상:** 메타인지 (자기 한계 인식), 능력 천장 vs 의지 천장 분리

채점 기준:
- 보고된 어려운 점과 실제 수행이 저하된 구간의 일치도
- "모르겠다"를 정확히 말한 경우 vs 모르면서 아는 척한 경우
- 해결하지 못한 이유에 대한 귀인 (능력 부족 vs 시간 부족 vs 동기 부족)과 로그 증거의 일치도

## 4.4 비언어적 대체 경로

### 4.4.1 필요성

출력 채널 독립성 원칙(제1절)에 따라, 언어적 표현 능력이 인지 능력 평가에 혼입되어서는 안 된다. 따라서 언어적 인터뷰를 수행할 수 없거나 언어적 인터뷰가 인지 능력을 왜곡할 위험이 있는 응시자를 위한 대체 경로가 필요하다.

### 4.4.2 대상

- 언어 장애가 있는 응시자
- 시험 언어가 모국어가 아닌 응시자
- 사회적 불안이 인터뷰 수행에 체계적 영향을 미치는 응시자
- 연령이 낮아 언어적 자기보고 능력이 충분히 발달하지 않은 응시자

### 4.4.3 대체 방법

| 방법 | 설명 | 한계 |
|------|------|------|
| 선택형 회고 | 로그의 특정 시점에 대해 복수의 설명 중 하나를 선택 | 설문의 한계를 일부 상속하지만, 로그 기반이므로 구체성 유지 |
| 시각적 재구성 | 과제 수행 영상을 보면서 "이 시점에서 주의를 기울인 곳"을 시각적으로 표시 | 메타인지의 시각적 측면만 포착 |
| 행동 재수행 | 동일 구조 변이에서 특정 구간을 다시 수행하게 한 후, 첫 수행과의 차이를 분석 | 학습 효과 통제 필요 |
| 보조 인터뷰어 | 수어 통역, 번역 등을 통한 중개 인터뷰 | 중개 과정에서의 정보 손실 |

### 4.4.4 대체 경로의 제약

비언어적 대체 경로는 언어적 인터뷰보다 메타인지 측정의 해상도가 낮을 수밖에 없다. 이 해상도 차이는 최종 점수 산출에서 메타인지 차원의 신뢰 구간을 넓히는 방식으로 반영한다. 즉, 비언어적 경로를 사용한 응시자의 메타인지 점수는 "정확하게 모른다"는 의미에서 불확실성이 더 크다.

## 4.5 채점자 신뢰도

### 4.5.1 요구 수준

인터뷰 채점의 채점자 간 신뢰도(inter-rater reliability) 목표:

- Cohen's κ ≥ 0.80 (질문 유형별)
- ICC (Intraclass Correlation Coefficient) ≥ 0.85 (전체 인터뷰 점수)

### 4.5.2 확보 방법

- **구조화된 채점 루브릭:** 각 질문 유형에 대해 0~5점 채점 기준을 사전 정의. 채점 기준에는 구체적인 응답 예시를 포함.
- **채점자 훈련:** 채점자는 인증 과정을 이수해야 하며, 훈련용 인터뷰 세트에서 기준 점수와의 일치도가 κ ≥ 0.80에 도달해야 인증.
- **이중 채점:** 모든 인터뷰는 최소 2인이 독립적으로 채점하며, 불일치가 발생하면 제3의 채점자가 최종 판정.
- **로그 기반 객관화:** 인터뷰 응답의 채점 기준 중 상당 부분이 "로그와의 일치도"이므로, 주관적 판단이 개입하는 영역이 제한됨.

### 4.5.3 인터뷰 시간

표준 인터뷰 시간은 **15~30분**이다. 질문은 로그에서 자동 생성된 5~10개의 주목 포인트에 대해 각 2~3분씩 수행한다.

## 4.6 인터뷰와 로그의 관계

인터뷰 데이터는 독립적인 점수를 산출하지 않는다. 인터뷰 데이터는 제5절의 **인터뷰 보정(Interview Calibration)** 과정에서 로그 기반 프로세스 메트릭과 교차 분석되며, 그 결과가 메타인지 차원의 점수에 반영된다.

핵심 원칙: **로그와 인터뷰가 불일치하면, 로그가 우선한다.** 단, 불일치 자체가 메타인지 점수에 대한 정보를 제공한다 (불일치가 클수록 메타인지 정확도가 낮음).
