# CPAS 설계명세서 — 제2절: 시스템 설계

> **Cognitive Process Assessment System (CPAS)**
> 본 문서는 CPAS 설계명세서 제2절 "시스템 설계"를 독립 문서로 분기하여 상세화한 것이다.

---

## 2.1 마이크로월드 (Microworld)

### 2.1.1 정의

마이크로월드란 **제한된 규칙이 지배하는 시뮬레이션 환경**이다. 응시자는 이 환경 안에서 정보를 탐색하고, 행동을 수행하고, 결과를 관찰하며, 전략을 수정한다.

마이크로월드는 현실 세계의 축소판이 아니다. 현실 세계의 복잡성을 통제하되, **인지적 도전의 구조**는 보존한 인공 환경이다.

### 2.1.2 세 가지 필수 속성

| 속성 | 설명 | 측정 기여 |
|------|------|-----------|
| 불완전 정보 | 모든 정보가 처음부터 보이지 않는다. 응시자가 능동적으로 열어봐야 확인할 수 있다. | "어떤 정보를 어떤 순서로 열어봤는가"가 사고의 탐색 트리를 외부화한다. |
| 동적 변화 | 환경은 응시자의 행동과 무관하게 자체적으로 변화한다. 응시자의 행동이 환경을 변화시키기도 한다. | 적응적 재계획, 변화 탐지 능력을 측정한다. |
| 다중 경로 | 목표에 도달하는 경로가 하나가 아니다. 최적 경로, 차선 경로, 비효율적 경로가 공존한다. | 수렴적 사고, 전략 선택의 질을 비교 평가한다. |

### 2.1.3 정보 은닉 구조

예를 들어 8개 변수가 있는 시스템에서 3개만 기본 노출되고, 나머지 5개는 클릭해서 확인해야 한다면:

- 어떤 변수를 먼저 열어봤는가 → 탐색 우선순위 (발산적 사고의 방향성)
- 열어보지 않은 변수가 있는가 → 탐색 공간 커버리지 (발산적 사고의 폭)
- 같은 변수를 반복해서 열어봤는가 → 작업기억 용량과 불확실성 관리
- 특정 변수를 열어본 직후 다른 특정 변수를 열어보는 패턴 → 변수 간 관계에 대한 가설 형성

**핵심 설계 원칙:** "깊이 있게 생각하지 않으면 절대 접근하지 않을 정보"가 반드시 존재해야 한다. 이것이 사고 깊이(Reasoning Depth)의 프록시가 된다. 1~2단계 추론만으로는 존재 자체를 인식하지 못하지만, 4~5단계 추론을 거치면 "저 정보를 확인해봐야 한다"는 결론에 도달하게 되는 숨겨진 정보 층위를 과제 안에 배치한다.

### 2.1.4 마이크로월드 규칙 세트

마이크로월드의 규칙 세트는 다음을 정의한다:

- **상태 공간:** 환경이 가질 수 있는 모든 상태의 집합
- **행동 공간:** 응시자가 수행할 수 있는 모든 행동의 집합
- **전이 함수:** 현재 상태 + 응시자 행동 → 다음 상태
- **관측 함수:** 현재 상태에서 응시자에게 보이는 정보의 범위
- **목표 함수:** 달성해야 할 목표의 정의와 달성 기준
- **시간 제약:** 과제의 총 제한 시간, 개별 행동의 제한 시간(있는 경우)

규칙 세트는 고정되지만, 초기 조건(변수의 초기값, 정보의 배치, 환경의 초기 상태)은 무작위화된다. 이것이 **구조 변이(Structural Variant)**의 기반이다.

## 2.2 시간 스케일별 과제 설계

CPAS는 네 개의 시간 스케일에서 서로 다른 인지 기제를 분리하여 측정한다. 시간 스케일의 분리는 과제 설계로 강제한다.

### 2.2.1 즉각 (Immediate): 0~3초

**측정 대상:** 패턴 인식, 직관적 판단, 반응 속도, 충동 억제

자극이 주어진 후 즉시 반응해야 하는 과제다. 숙고할 시간이 없으므로, 자동화된 인지 프로세스와 직관의 질이 드러난다.

과제 특성:
- 자극 제시 후 반응 시간이 3초 이내로 제한
- 정확도와 반응 시간의 조합으로 채점
- 충동적 반응(빠르지만 부정확)과 숙련된 직관(빠르고 정확)의 구분이 핵심
- 반복 시행을 통해 우연과 실력을 통계적으로 분리

### 2.2.2 단기 (Short-term): 10~60초

**측정 대상:** 발산-수렴 사이클 1회, 제한 시간 내 최적 경로 탐색

하나의 문제를 해결하기에 충분하지만 여러 전략을 시도하기에는 부족한 시간이 주어진다. 발산적 사고(경우의 수 탐색)와 수렴적 사고(최적 선택)가 한 사이클 돌아가는 규모다.

과제 특성:
- 정보 탐색 → 가설 형성 → 행동 선택의 단일 사이클
- 탐색 효율성(불필요한 정보를 건너뛰는 능력)이 핵심 메트릭
- 시간 압박 하에서의 판단 질 측정

### 2.2.3 중기 (Medium-term): 3~10분

**측정 대상:** 다단계 계획, 하위 목표 설정, 자원 배분, 중간 피드백 반영

복수의 하위 문제를 순서대로 또는 병렬로 해결해야 하는 과제다. 중간 결과에 따라 이후 전략을 수정해야 하므로, 적응적 재계획 능력이 드러난다.

과제 특성:
- 하위 목표가 2~5개 존재
- 하위 목표 간 자원 경쟁 (시간, 주의, 정보 접근 횟수 제한 등)
- 환경이 중간에 1~2회 변화하여 기존 계획의 수정을 요구
- 메타인지 지표: 전략 전환 시점이 최적 시점과 얼마나 가까운가

### 2.2.4 장기 (Long-term): 30분~수 시간

**측정 대상:** 전략 수정, 학습 곡선, 메타인지적 조절, 인지적 지구력

장시간에 걸쳐 복잡한 문제를 해결해야 하는 과제다. 시간이 지남에 따라 응시자의 전략이 어떻게 진화하는지, 초기 실패에서 무엇을 학습하는지, 인지적 피로에 어떻게 대응하는지가 드러난다.

과제 특성:
- 초기에는 불완전한 이해로 시작하여 점진적으로 환경 모델을 구축
- 같은 유형의 하위 문제가 반복되되 난이도가 점진적으로 상승
- 학습 곡선의 기울기 = 학습 능력의 프록시
- 인지적 몰입 의지(차원 9) 측정의 핵심 구간
- 인지적 피로 시점에서의 행동 변화 패턴 관찰

## 2.3 1ms 틱 레이트 로깅

### 2.3.1 기본 전제

CPAS는 **1ms가 1틱 레이트를 완벽하게 보장한다는 가정** 하에 설계되었다.

### 2.3.2 로그 레코드 구조

매 틱마다 기록되는 로그 레코드의 최소 필드:

| 필드 | 설명 |
|------|------|
| timestamp | 과제 시작으로부터의 경과 시간 (ms 단위) |
| event_type | 행동 유형: 정보_조회, 행동_실행, 환경_관측, 무행동 |
| target_id | 행동 대상의 식별자 (조회한 변수 ID, 실행한 행동 ID 등) |
| context_state | 행동 시점의 환경 상태 스냅샷 (또는 스냅샷 참조 키) |
| input_channel | 사용된 입력 장치 (키보드, 마우스, 시선, 음성 등) |
| motor_duration | 물리적 조작에 소요된 시간 추정치 (ms) |
| cognitive_duration | 인지 처리 시간 추정치 (총 경과 시간 − 물리적 조작 시간) |

### 2.3.3 로깅 해상도의 의미

왜 1ms인가:

- **미세한 망설임을 포착한다.** 200ms의 지연과 800ms의 지연은 서로 다른 인지적 상태를 반영할 수 있다.
- **행동 클러스터링의 정밀도를 높인다.** 연속된 행동들이 하나의 전략적 단위를 이루는지 판단하려면, 행동 간 간격의 미세한 패턴 분석이 필요하다.
- **물리적 실행 시간과 인지 처리 시간의 분리 정밀도를 결정한다.** 해상도가 낮으면 두 시간을 분리하는 것이 물리적으로 불가능해진다.

## 2.4 구조 변이 (Structural Variant)

### 2.4.1 정의

구조 변이란 **동일한 규칙 세트와 인지적 도전 구조를 유지하면서, 표면적 요소(변수값, 배치, 시각적 외형)만 변경한 과제 인스턴스**를 말한다.

### 2.4.2 목적

- **과제 유출 방어:** 특정 과제 인스턴스의 "정답"이 유출되어도 다음 응시자는 표면이 다른 인스턴스를 받으므로 이점이 없다.
- **재응시 공정성:** 동일 응시자가 재응시할 때, 구조적으로 동등하지만 표면이 다른 인스턴스를 받는다.
- **규준 집단 구축:** 서로 다른 인스턴스를 받은 응시자들의 점수를 공정하게 비교하기 위해 IRT 모델이 인스턴스 간 난이도를 통계적으로 보정한다.

### 2.4.3 변이 가능 요소 vs 고정 요소

| 구분 | 항목 | 예시 |
|------|------|------|
| 고정 | 규칙 세트 | 상태 전이 함수, 행동 공간의 구조 |
| 고정 | 인지적 도전 구조 | 필요한 추론 단계 수, 정보 은닉의 층위 수 |
| 고정 | 최적 경로의 존재 | 최적 해가 존재한다는 사실 자체 |
| 변이 | 초기 상태 | 변수의 초기값, 객체의 초기 배치 |
| 변이 | 표면적 외형 | 색상, 레이블, 시각적 레이아웃 |
| 변이 | 최적 경로의 구체적 내용 | 어떤 행동 순서가 최적인지 |
| 변이 | 환경 변화 시점 | 동적 변화가 발생하는 타이밍 |

### 2.4.4 변이 생성 자동화

구조 변이는 규칙 세트를 파라미터화하고, 초기 조건을 난수로 생성하여 자동 생성한다. 생성된 각 인스턴스에 대해 다음을 자동 검증한다:

- 최적 해가 존재하는가
- 최적 해에 도달하는 데 필요한 추론 단계 수가 기준 범위 이내인가
- 해결 불가능한 교착 상태(deadlock)가 발생하지 않는가
- 난이도 추정치가 기준 범위 이내인가

## 2.5 과제 배터리: 7개 소재 유형

CPAS의 과제는 단일 유형이 아니다. 7개의 소재 유형으로 구성된 **과제 배터리(Task Battery)**를 사용하여 특정 소재에 대한 친숙도 편향을 최소화한다.

### 유형 1: 연구 과제형

**주요 측정 대상:** 시간 압박 없는 체계적 탐색과 추론 깊이

시간 제한이 느슨하거나 없는 상태에서, 복잡한 정보 체계를 탐색하고 규칙을 발견하는 과제다. 학술 연구의 가설 형성-검증 사이클과 구조적으로 유사하다.

- 장기 시간 스케일 과제의 핵심 소재
- 정보의 양이 많고 계층적으로 구조화
- "깊이 있게 생각하지 않으면 접근하지 않을 정보"를 다층적으로 배치
- 탐색의 체계성과 가설-검증 루프의 질을 측정

### 유형 2: 인간관계 시나리오형

**주요 측정 대상:** 의도 추론, 전략 유연성, 심리적 압박 하 판단력

복수의 가상 인물이 등장하는 사회적 상황에서 의사결정을 내리는 과제다. 각 인물의 의도와 전략을 추론해야 하며, 자신의 행동이 타인의 반응을 변화시킨다.

- 관점 전환(차원 7) 측정의 핵심 소재
- 동적 대화 트리 기반 — 응시자의 선택에 따라 대화가 분기
- 심리적 압박(시간 제한, 갈등 상황)을 의도적으로 설계
- 미해결 과제: 동적 대화 트리의 규모와 분기 복잡도 설계 (9.5절)

### 유형 3: 즉각 반응 게임형

**주요 측정 대상:** 패턴 인식, 반응 속도, 충동 억제

짧은 자극에 빠르게 반응해야 하는 게임 형식의 과제. 즉각 시간 스케일(0~3초) 전용.

- 반복 시행을 통해 통계적 안정성 확보
- Go/No-Go 패러다임 변형으로 충동 억제 측정
- 패턴의 복잡도를 점진적으로 높여 직관의 한계를 탐색
- 충동적 반응과 숙련된 직관의 분리가 설계의 핵심 도전 (7.5절)

### 유형 4: 사업 기반 프로젝트형

**주요 측정 대상:** 자원 배분, 우선순위 결정, 불확실성 하 의사결정

제한된 자원(시간, 예산, 인력)을 배분하여 복수의 목표를 달성해야 하는 프로젝트 시뮬레이션이다.

- 중기~장기 시간 스케일
- 자원 경쟁 구조가 트레이드오프 판단을 강제
- 불확실한 정보 하에서의 의사결정 질 측정
- 결과에 대한 피드백이 지연되어, 단기적 선택과 장기적 이득의 충돌을 유도

### 유형 5: 직관 측정형

**주요 측정 대상:** 과정이 압축된 내적 사고 (정확도-속도 조합)

의식적 추론 과정 없이 "느낌"으로 판단해야 하는 과제. 직관이란 과거 경험에서 압축된 패턴 인식이므로, 정확도와 속도의 조합이 직관의 질을 나타낸다.

- 즉각~단기 시간 스케일
- 정답의 근거를 설명할 수 없지만 정답을 맞히는 능력을 측정
- 인터뷰(제4절)에서 "왜 그렇게 판단했는가"에 대한 자기보고를 수집하여 메타인지와 교차 분석

### 유형 6: 교육 커리큘럼 기반형

**주요 측정 대상:** 연령대별 규준 확보를 위한 스케일링

초등학생부터 성인까지 적용할 수 있도록, 교육 과정에서 다루는 소재를 활용한 과제. 단, 지식 자체를 측정하지 않고 지식을 활용하는 **과정의 구조**를 측정한다.

- 규준 집단 구축의 핵심 도구
- 연령대별 난이도 스케일링이 가능한 과제 구조
- 지식이 아닌 과정을 측정한다는 원칙의 가장 엄격한 시험대

### 유형 7: 관점 전환형

**주요 측정 대상:** 다중 역할 수행, 시점 간 연결성 및 통합

하나의 상황을 복수의 역할/입장에서 수행하고, 각 역할에서의 판단을 통합하는 과제.

- 동일 마이크로월드에서 역할만 바꿔서 재수행
- 이전 역할에서의 탐색 경험이 현재 역할의 전략에 어떻게 반영되는지 측정
- 정치적 소재 사용 시의 안전성 문제 (9.4절)
- 관점 간 연결성의 채점 기준 표준화가 미해결 (9.5절)

## 2.6 입력 인터페이스 독립성

### 2.6.1 원칙

제1절의 출력 채널 독립성 원칙을 시스템 설계 수준에서 구현한 것이다. 모든 마이크로월드는 **추상적 행동 인터페이스(Abstract Action Interface)**를 통해 응시자와 소통한다. "무엇을 하겠는가"(인지적 선택)만 전달하며, "어떤 물리적 동작으로 하겠는가"(운동 실행)는 별도의 입력 어댑터가 처리한다.

### 2.6.2 입력 어댑터 구조

```
[응시자] → [입력 장치] → [입력 어댑터] → [추상 행동 인터페이스] → [마이크로월드]
                              ↓
                    [운동 시간 분리 기록]
```

입력 어댑터의 역할:
- 물리적 입력(클릭, 키 입력, 시선 고정, 음성 명령)을 추상적 행동으로 변환
- 물리적 조작에 소요된 시간(운동 시간)을 분리하여 기록
- 입력 장치별 특성(지연 시간, 정밀도)을 보정

### 2.6.3 지원 입력 장치 및 보정

| 입력 장치 | 운동 시간 분리 방법 | 비고 |
|-----------|---------------------|------|
| 마우스/키보드 | 클릭/키 입력 시간 − 인지 처리 시간 추정 | 가장 보편적 |
| 터치스크린 | 터치 지점 도달 시간 분리 | 모바일 환경 지원 |
| 시선 추적 | 고정(fixation) 시작 시점 = 인지적 선택 시점으로 추정 | 순수 인지 시간에 가장 근접 |
| 음성 입력 | 발화 시작~종료 시간 분리 | 신체 장애 대응 |
| BCI | 의도 신호 탐지 시점 = 인지적 선택 시점 | 미래 기준 채널 후보 |

시선 추적이나 BCI를 통해 측정된 순수 인지 처리 시간을 기준으로, 다른 입력 장치의 보정 계수를 산출한다. 보정 계수는 응시자별, 입력 장치별로 캘리브레이션 과제를 통해 사전 측정한다.

## 2.7 과제 시퀀스 설계

### 2.7.1 배치 순서 원칙

7개 소재 유형 × 4개 시간 스케일의 유효 조합으로 과제 슬롯을 구성하며, 배치 순서는 다음 원칙을 따른다:

- **인지적 워밍업:** 가장 먼저 즉각/단기 과제를 배치하여 환경 적응 유도
- **피로 관리:** 중기/장기 과제 사이에 짧은 즉각 과제를 배치하여 인지적 리셋 유도
- **순서 효과 통제:** 과제 배치 순서를 응시자 간에 무작위화하되, 워밍업과 피로 관리 원칙은 유지
- **시간 스케일 간 독립성:** 한 시간 스케일의 과제 결과가 다른 시간 스케일에 체계적 영향을 미치지 않도록 설계

### 2.7.2 총 소요 시간

전체 과제 배터리의 표준 소요 시간 목표 범위는 **3~5시간**이다. 장기 시간 스케일 과제가 전체 시간의 대부분을 차지한다.
