# CPAS 설계명세서 — 제5절: 메커니즘

> **Cognitive Process Assessment System (CPAS)**
> 본 문서는 CPAS 설계명세서 제5절 "메커니즘"을 독립 문서로 분기하여 상세화한 것이다.

---

## 5.1 로그 분석 파이프라인 (8단계)

1ms 틱 로그에서 종합 점수까지의 데이터 변환 과정을 8단계로 정의한다.

### 단계 1: 원시 로그 수집 (Raw Log Collection)

마이크로월드에서 생성된 모든 이벤트를 타임스탬프와 함께 수집한다. 이 단계에서는 어떤 필터링이나 가공도 하지 않는다. 원시 로그의 무결성이 이후 모든 분석의 기반이다.

수집 항목: timestamp, event_type, target_id, context_state, input_channel, motor_duration 등 (제2절 로그 레코드 구조 참조)

### 단계 2: 운동 시간 분리 (Motor Time Separation)

출력 채널 독립성 원칙에 따라, 각 행동에서 물리적 조작에 소요된 시간(운동 시간)을 분리한다.

- 입력 장치별 캘리브레이션 데이터를 적용하여 운동 시간 추정
- 총 경과 시간에서 운동 시간을 빼서 순수 인지 처리 시간 산출
- 입력 장치별 보정 계수 적용

### 단계 3: 이벤트 시퀀스 정규화 (Event Sequence Normalization)

서로 다른 입력 장치, 서로 다른 구조 변이 인스턴스에서 생성된 로그를 동일한 표현 체계로 변환한다.

- 구체적인 변수 ID를 추상적 역할 ID로 매핑 (예: "변수 C" → "2번째로 은닉된 정보 노드")
- 절대 시간을 상대 시간(과제 시작 후 경과 시간)으로 변환
- 입력 장치별 차이를 제거한 추상적 행동 시퀀스 생성

### 단계 4: 행동 세그멘테이션 (Action Segmentation)

연속된 행동 스트림을 의미 있는 단위로 분절한다.

- **행동 클러스터 식별:** 행동 간 간격이 임계값 이하인 연속 행동들을 하나의 클러스터로 묶음
- **발산-수렴 구간 식별:** 탐색 폭이 확장되는 구간(발산)과 축소되는 구간(수렴)을 자동 탐지
- **전략 전환점 식별:** 행동 패턴이 급격히 변하는 시점을 변화점 탐지(change-point detection) 알고리즘으로 식별
- **환경 변화 대응 구간 식별:** 환경 변화 이벤트 전후의 행동을 별도 세그먼트로 분리

### 단계 5: 프로세스 메트릭 추출 (Process Metric Extraction)

세그멘테이션된 행동 데이터에서 8개 프로세스 메트릭(제3절)을 계산한다.

각 메트릭에 대해:
- 과제별 원시 값 산출
- 시간 스케일별(즉각/단기/중기/장기) 분리 산출
- 과제 소재 유형별(7종) 분리 산출
- 신뢰 구간 추정

### 단계 6: 계단 도달 판정 (Step-Reach Determination)

각 과제에서 응시자가 도달한 최대 계단을 자동으로 판정한다.

- 계단별 도달 기준은 과제 설계 시 사전 정의
- 도달 여부는 행동 로그에서 객관적으로 판정 가능해야 함
- 능력 천장과 의지 천장을 분리하기 위한 포기 패턴 분류

### 단계 7: 인터뷰 보정 (Interview Calibration)

인터뷰 데이터를 프로세스 메트릭과 교차 분석하여 메타인지 차원의 점수를 산출한다. (5.2절에서 상세화)

### 단계 8: 종합 점수 산출 (Composite Score Computation)

9개 인지 차원의 개별 점수를 종합하여 단일 점수를 산출한다. (5.3절에서 상세화)

## 5.2 인터뷰 보정: 일치-불일치 매트릭스

### 5.2.1 매트릭스 구조

인터뷰 보정의 핵심 도구는 **일치-불일치 매트릭스(Concordance-Discordance Matrix)**다. 이 매트릭스는 행동 로그에서 추론한 인지 상태와 인터뷰에서 보고된 인지 상태를 2×2로 교차한다.

|  | 로그: 체계적 행동 | 로그: 비체계적 행동 |
|--|-------------------|---------------------|
| **보고: 체계적 사고** | 일치-긍정 (TP) | 불일치-과대보고 (FP) |
| **보고: 비체계적 사고** | 불일치-과소보고 (FN) | 일치-부정 (TN) |

### 5.2.2 각 셀의 해석

- **일치-긍정 (TP):** 체계적으로 행동했고, 본인도 그것을 정확히 인식. 메타인지 정확도 높음.
- **불일치-과대보고 (FP):** 비체계적으로 행동했지만, 사후에 체계적이었다고 합리화. 메타인지 정확도 낮음. 사후 합리화 편향의 증거.
- **불일치-과소보고 (FN):** 체계적으로 행동했지만, 본인은 그것을 인식하지 못함. 무의식적 유능함. 메타인지 정확도는 낮지만, 인지 능력 자체는 높음.
- **일치-부정 (TN):** 비체계적으로 행동했고, 본인도 그것을 정확히 인식. 자기 한계 인식 능력 높음.

### 5.2.3 메타인지 점수 산출

메타인지 점수는 매트릭스에서 다음과 같이 산출한다:

- **정확도 (Accuracy):** (TP + TN) / 전체. 자기 인식이 실제와 일치하는 비율.
- **과신 지수 (Overconfidence Index):** FP / (FP + TN). 실제보다 자기를 과대평가하는 경향.
- **겸손 지수 (Humility Index):** FN / (FN + TP). 실제보다 자기를 과소평가하는 경향.
- **캘리브레이션 지수:** 확신도 질문(유형 3)의 확신도-정확도 상관 계수.

## 5.3 점수 산출

### 5.3.1 g-factor 추출

9개 인지 차원의 개별 점수 간 상관 구조에서 일반 인지 인자(g-factor)를 추출한다. 이는 Spearman의 일반 지능 인자 개념과 유사하지만, 측정 기반이 전통적 IQ 테스트가 아닌 프로세스 메트릭이라는 점에서 다르다.

방법:
- 규준 집단 데이터에서 9개 차원 점수의 공분산 행렬을 산출
- 주성분 분석(PCA) 또는 탐색적 요인 분석(EFA)으로 제1인자(g)를 추출
- g-factor에 대한 각 차원의 인자 적재량(factor loading)이 곧 종합 점수에서의 가중치가 됨

**핵심 원칙:** 가중치는 설계자가 정하지 않는다. 데이터의 공분산 구조에서 통계적으로 도출한다. 규준 집단이 바뀌면 가중치도 자연스럽게 재조정된다.

### 5.3.2 IRT 기반 능력 추정

각 프로세스 메트릭에 대해 문항 반응 이론(Item Response Theory)을 적용한다.

- 각 과제 인스턴스(구조 변이)를 하나의 "문항"으로 취급
- 각 메트릭의 성과 수준을 "응답"으로 취급
- 2-모수 IRT 모델(2PL)을 적용하여 과제 난이도와 응시자 능력을 동시 추정
- 서로 다른 구조 변이를 받은 응시자들의 점수를 공정하게 비교 가능

### 5.3.3 규준 기반 표준 점수

IRT로 추정된 능력치를 규준 집단에 대한 상대적 위치로 변환한다.

- 연령대별, 교육 수준별 규준 집단을 구축
- 능력치를 표준 점수(z-score 또는 T-score)로 변환
- 종합 점수는 9개 차원의 가중 합산을 표준화한 단일 숫자

### 5.3.4 점수의 불확실성 표현

각 차원의 점수와 종합 점수에 대해 **신뢰 구간**을 함께 산출한다.

- 표준 오차(SE)는 IRT 모델에서 자동 산출
- 비언어적 대체 경로를 사용한 경우 메타인지 차원의 SE가 증가
- 과제 수가 적은 시간 스케일에서 해당 차원의 SE가 증가

## 5.4 재검사 신뢰도

### 5.4.1 목표

재검사 신뢰도(test-retest reliability) 목표: **r ≥ 0.85**

### 5.4.2 측정 방법

- 동일 응시자에게 구조적으로 동등하지만 표면이 다른 과제(구조 변이)를 **2주~4주 간격**으로 반복 실시
- 두 시점의 종합 점수 간 Pearson 상관 계수를 산출
- 인지 차원별 재검사 신뢰도도 개별 산출

### 5.4.3 신뢰도 저해 요인과 대응

| 저해 요인 | 설명 | 대응 |
|-----------|------|------|
| 연습 효과 | 두 번째 시행에서 과제 구조에 대한 학습이 일어남 | 구조 변이로 표면을 바꿔 연습 이점 최소화 |
| 컨디션 변동 | 신체적/정서적 상태가 달라짐 | 컨디션 보정 (7.8절), 재시험 제도 (8절) |
| 성장/퇴행 | 측정 간격 동안 실제 인지 능력이 변화 | 간격을 2~4주로 제한하여 단기 변화 최소화 |
| 동기 변동 | 첫 번째와 두 번째 시행의 동기 수준이 다름 | 인지적 몰입 의지 차원으로 분리 감지 |

## 5.5 타당도 검증 전략

### 5.5.1 구성 타당도 (Construct Validity)

9개 인지 차원이 실제로 서로 다른 구성 개념을 측정하는지 확인한다.

- 확인적 요인 분석(CFA)으로 9인자 모델의 적합도 검증
- 차원 간 상관이 0.3~0.7 범위: 관련되어 있지만 독립적인 구성 개념
- 차원 간 상관이 0.9 이상: 두 차원이 동일한 것을 측정할 가능성 → 차원 통합 검토

### 5.5.2 예측 타당도 (Predictive Validity)

CPAS 점수가 실제 인지적 수행을 예측하는지 확인한다.

- 학업 성취, 직무 수행, 일상적 문제 해결 등 외적 기준과의 상관
- 기존 IQ 테스트와의 수렴 타당도 (어느 정도 상관이 있어야 하지만, 너무 높으면 CPAS의 추가 가치가 없음)
- 기존 IQ 테스트와의 증분 타당도: CPAS가 IQ를 넘어서 추가적으로 설명하는 분산의 크기

### 5.5.3 변별 타당도 (Discriminant Validity)

CPAS가 측정하지 않기로 한 것(지식, 성격, 정서 지능 등)과 충분히 낮은 상관을 보이는지 확인한다.
