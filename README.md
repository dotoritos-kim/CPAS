# CPAS — Cognitive Process Assessment System

**프로세스 기반 종합지능 측정 시스템**

기존 평가 도구(IQ 테스트, 수능, 설문)는 정답 여부만 측정한다. CPAS는 사고의 과정 자체를 측정한다. 응시자를 통제된 시뮬레이션 환경(마이크로월드)에 배치하고, 1ms 틱 레이트로 모든 행동을 기록하며, 구조화된 인터뷰로 로그의 모호성을 검증하여 종합 점수 하나를 산출한다.

---

## 핵심 원리

**측정 도구가 아니라 측정 환경을 설계한다.** 사고는 직접 관측할 수 없으므로, 사고가 외부화될 수밖에 없는 조건을 만들고 그 흔적을 읽는다. 응시자는 목표를 달성하려고 할 뿐이고, 시스템이 과정을 기록한다.

**인지 측정은 출력 채널에 독립적이어야 한다.** 키보드, 마우스, 시선 추적, 음성 입력, BCI 등 어떤 입력 장치를 사용하든 로그에 기록되는 인지적 선택의 구조는 동일하다. 물리적 조작 시간은 인지 처리 시간에서 분리된다.

---

## 측정 차원 (9개)

| 차원 | 설명 |
|------|------|
| 발산적 사고 | 경우의 수 탐색의 폭과 체계성 |
| 수렴적 사고 | 최적 선택지를 찾아내는 정확도 |
| 사고의 깊이 | 추론 체인의 단계 수 |
| 분기 간 연관성 | 사고 분기들의 유기적 연결 |
| 적응적 재계획 | 상황 변화에 대한 대응 속도 |
| 메타인지 | 자기 사고 과정의 인식 정확도 |
| 관점 전환 | 다중 시점 운용 및 통합 능력 |
| 비판적 사고 | 모순, 궤변, 허위 정보 탐지 |
| 인지적 몰입 의지 | 필요한 깊이까지 도달하는 저항 없는 진입과 적절한 중단 |

## 문서 구조

[CPAS_Design_Specification.md](./docs/CPAS_Design_Specification.md) 는 9개 본문 절과 부록으로 구성된다.

| 절 | 제목 | 내용 |
|----|------|------|
| 1 | 시스템 의도 | 목적, 측정 대상 9개 차원, 핵심 철학 및 출력 채널 독립성 원칙 |
| 2 | 시스템 설계 | 마이크로월드, 시간 스케일(즉각/단기/중기/장기), 1ms 로깅, 구조 변이, 과제 배터리(7개 소재 유형), 입력 인터페이스 독립성 |
| 3 | 평가 항목 | 프로세스 메트릭 8종, 계단 도달 측정, 능력 천장/의지 천장 분리, 결과 메트릭 |
| 4 | 인터뷰 | 로그 기반 구조화 인터뷰, 역할 정의(보조 도구), 질문 유형 5종, 비언어적 대체 경로, 채점자 신뢰도 |
| 5 | 메커니즘 | 로그 분석 파이프라인(8단계), 인터뷰 보정(일치-불일치 매트릭스), 점수 산출(g-factor/IRT/규준 기반), 재검사 신뢰도 |
| 6 | 최종 평가 | 종합 점수 단일 산출물, 점수의 예측적 의미, 응시자 경험 설계 |
| 7 | 맹점 | 사고 깊이 추론 한계, 생태학적 타당도, 언어 편향, 신경다양성, 충동/숙련 구분, 문화적 편향, 신체 조건, 컨디션 (8항목) |
| 8 | 항의 및 재시험 | 재응시 왜곡 방어(구조 변이/이상 탐지/엘리트 구분/인터뷰 필터), 항의 사유, 재시험 절차, 사전 연기 제도 |
| 9 | 보완 과제 | 함정 정보 탐지, 궤변 탐지(유형 A~D), 지식/지혜 분리, 관점 전환 및 정치적 소재, 미해결 설계 난제 7항목 |
| 부록 | 용어 정의 | 전체 문서에서 사용된 전문 용어 30항목 정의 |

## 과제 소재 유형 (7종)

| 유형 | 주요 측정 대상 |
|------|---------------|
| 연구 과제형 | 시간 압박 없는 체계적 탐색과 추론 깊이 |
| 인간관계 시나리오형 | 의도 추론, 전략 유연성, 심리적 압박 하 판단력 |
| 즉각 반응 게임형 | 패턴 인식, 반응 속도, 충동 억제 |
| 사업 기반 프로젝트형 | 자원 배분, 우선순위, 불확실성 하 의사결정 |
| 직관 측정형 | 과정 압축된 내적 사고 (정확도-속도 조합) |
| 교육 커리큘럼 기반형 | 연령대별 규준 확보를 위한 스케일링 |
| 관점 전환형 | 다중 역할 수행, 시점 간 연결성 및 통합 |

## 설계 전제

- 1ms가 1틱 레이트를 완벽하게 보장한다는 가정 하에 설계됨
- 종합 점수는 여러 차원의 상대값 합산이며, 응시자에게는 단일 숫자로 전달
- 가중치는 설계자가 정하지 않고, 규준 집단 데이터의 공분산 구조에서 통계적으로 도출
- 과제 구조를 암기해도 재응시에서 이점을 얻기 어렵도록 구조 변이와 이상 탐지를 적용

---

## 미해결 과제 요약

명세서 9.5절에 정리된 선행 해결 과제:

- 규칙 모호성과 채점 안정성의 균형 (궤변 과제 설계)
- "이득"의 다층적 정의 (가치 판단 vs 객관적 측정)
- 유형 D 정보의 과제 내 검증 경로 보장
- 함정 정보와 구조 변이의 상호작용 보정
- 인간관계 시나리오의 동적 대화 트리 설계
- 관점 전환 과제의 정치적 안전성과 측정 깊이 균형
- 관점 간 연결성의 채점 기준 표준화
